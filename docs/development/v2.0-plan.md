## 概要

v2.0 はセマンティックバージョニング上の **メジャー破壊的変更リリース** である。
v1.3.x/v1.4.0 で段階的に準備してきた以下の方向性を完遂する:

1. **static 依存の排除** -- PuaTokenMapper インスタンス化、テスト間状態リーク根絶
2. **MultilingualPhonemizer の Strategy パターン完全移行** -- `ILanguageG2PHandler` による言語プラグイン化
3. **piper-plus データモデル互換** -- `int[]` PhonemeIdMap、Prosody フラット配列、pua.json ランタイム読み込み
4. **設定・API の整理** -- ValidatedPiperConfig ネスト化、IPiperConfigReadOnly、Obsolete 削除

全16項目、5マイルストーン、見積もり約37人日（チケット合計29人日 + 統合バッファ・補正8人日）。
4フェーズに分類し、フェーズ内は並行実施可能、フェーズ間は依存順に実施する。

### 実装完了サマリー（2026-04-09）

全4フェーズ・16チケットの実装が完了した:

- **Phase 1（基盤リファクタリング）**: P1-1 ~ P1-6 の6チケット完了。PuaTokenMapper インスタンス化、ILanguageG2PHandler 全面移行、pua.json ランタイム読み込み、IPhonemizerBackend 廃止、Obsolete 削除を実施。
- **Phase 2（データモデル互換性）**: P2-1 ~ P2-3 の3チケット完了。PhonemeIdMap int[] 型変更、Prosody フラット配列化、NativeArray 統一を実施。
- **Phase 3（設定・API 整理）**: P3-1 ~ P3-6 の6チケット完了。ValidatedPiperConfig ネスト構造化、IPiperConfigReadOnly、Validate() 純粋関数化、BackendSelector/AudioNormalizer 切り出し、SynthesisRequest public 昇格を実施。
- **Phase 4（言語検出高度化）**: P4-1 完了。N-gram 言語検出（HybridLanguageDetector）を導入。

## 不採用項目

| 提案 | 理由 |
|------|------|
| public API 戻り値 `IMemoryOwner<float>` 化 | `AudioClip.SetData` が `float[]` を要求するため変換コストが発生。費用対効果が見合わない。NativeArray 統一（P2-3）で中間コピーは解消可能 |
| IWarmupStrategy インターフェース | YAGNI。現実的なバリエーションがダミー入力パラメータ違いのみ（v1.3.x で不採用決定済み） |
| AudioBufferPool (TTS 専用プール) | `ArrayPool.Shared` で十分。TTS 呼び出し頻度はリアルタイムレンダリングほどではない（v1.3.x で不採用決定済み） |
| Span/Memory ベース public API | Unity API が `float[]` を要求するため変換コストが発生（v1.3.x で不採用決定済み） |
| Builder パターン (PiperConfig) | Unity の `[Serializable]` + Inspector 編集と相性が悪い（v1.3.x で不採用決定済み） |
| ISynthesisPipeline 抽象 | piper-plus も不採用。YAGNI。ストリーミング対応時に再評価（v1.3.x で不採用決定済み） |
| 言語ヒント API `[sv]` タグ | `defaultLatinLanguage` 切り替えで代替可能（v1.3.x で不採用決定済み） |
| Phoneme ID `int` → `long` 統一 | Sentis が int32 テンソル要求。C# 側で long を使うメリットなし（v1.3.x で不採用決定済み） |
| MultilingualPhonemizer 全同期化（async 完全廃止） | WebGL の `DotNetG2PPhonemizer.InitializeAsync()` が非同期必須。メソッドシグネチャは async 維持が安全 |

## マイグレーションガイド概要

### v1.x → v2.0 で必要な変更

#### 1. PuaTokenMapper 参照の更新

```csharp
// v1.x (static)
var ch = PuaTokenMapper.Register("ch");
var mapped = PuaTokenMapper.MapSequence(tokens);

// v2.0 (instance)
var mapper = new PuaTokenMapper();          // or DI
var ch = mapper.Register("ch");
var mapped = mapper.MapSequence(tokens);
```

#### 2. MultilingualPhonemizer コンストラクタ

```csharp
// v1.x (Obsolete constructor -- 削除)
var mp = new MultilingualPhonemizer(languages, "en", jaPhonemizer, ...);

// v2.0 (Options のみ)
var options = new MultilingualPhonemizerOptions
{
    Languages = languages,
    DefaultLatinLanguage = "en",
    // 個別エンジンプロパティ → ILanguageG2PHandler 登録に移行
};
var mp = new MultilingualPhonemizer(options);
```

#### 3. PhonemeIdMap 型変更

```csharp
// v1.x
Dictionary<string, int> phonemeIdMap = voiceConfig.PhonemeIdMap;
int id = phonemeIdMap["a"];

// v2.0
Dictionary<string, int[]> phonemeIdMap = voiceConfig.PhonemeIdMap;
int[] ids = phonemeIdMap["a"];  // 通常は ids.Length == 1
```

#### 4. ValidatedPiperConfig アクセスパス

```csharp
// v1.x
var backend = validatedConfig.Backend;
var sampleRate = validatedConfig.SampleRate;

// v2.0
var backend = validatedConfig.Inference.Backend;
var sampleRate = validatedConfig.Audio.SampleRate;
```

#### 5. IPhonemizerBackend 廃止

```csharp
// v1.x (テストスタブ)
var options = new MultilingualPhonemizerOptions
{
    EnPhonemizer = mockBackend,  // IPhonemizerBackend
};

// v2.0 (ILanguageG2PHandler)
var options = new MultilingualPhonemizerOptions
{
    Handlers = new Dictionary<string, ILanguageG2PHandler>
    {
        ["en"] = new MockEnglishHandler(),
    },
};
```
