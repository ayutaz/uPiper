```

  マイグレーションガイド完成

```

### グループ内の順序制約

**グループ A（データモデル）**: P2-1 と P2-2 は独立して実施可能だが、同一エージェントが順次実施する方が効率的。P2-1 の `PiperVoiceConfig.PhonemeIdMap` 型変更がP2-2 のテストコードにも影響するため。

推奨順序: **P2-1 → P2-2**

**グループ B（Config 三部作）**: P3-1 → P3-2 / P3-3 の順序制約あり。P3-2 と P3-3 は P3-1 完了後に並行可能だが、3つとも `ValidatedPiperConfig.cs` を変更するため同一エージェントが順次実施を推奨。

推奨順序: **P3-1 → P3-3 → P3-2**（P3-1 設計ドキュメントの推奨に準拠）

**グループ C（ロジック切り出し）**: P3-4 と P3-5 は互いに独立。並行可能だが規模が小さいため同一エージェントで順次実施。

推奨順序: **P3-5 → P3-4**（P3-5 がシンプルなためパターン確立に有用）

### タスク詳細

**P3-2（0.5 人日）**: `IPiperConfigReadOnly` インターフェースを定義。`ValidatedPiperConfig` が実装。`TTSSynthesisOrchestrator` の依存型を `IPiperConfigReadOnly` に変更。`PiperTTS._validatedConfig` のフィールド型変更。
