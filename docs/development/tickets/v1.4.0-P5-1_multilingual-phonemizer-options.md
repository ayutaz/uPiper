**ステータス**: 未着手

## 1. タスク目的とゴール

`MultilingualPhonemizer` のコンストラクタ（L96-106）は現在10パラメータを受け取っており、新言語追加のたびにパラメータが増加する。Optionsオブジェクトを導入し、コンストラクタの引数をグループ化する。

**ゴール**:
- `MultilingualPhonemizerOptions` クラスを新規作成
- `MultilingualPhonemizer` が Options を受け取る新コンストラクタを持つ
- 旧コンストラクタに `[Obsolete]` を付与して1バージョン猶予
- MS3-1（スウェーデン語追加）時に Options にプロパティ1つ追加するだけで済む拡張性を確保
- 全テストパス

**MS3-1先行実施の利点**: スウェーデン語追加時に `svEngine` パラメータを旧コンストラクタに追加する必要がなくなり、Options にプロパティを1つ追加するだけで対応可能。

## 2. 実装する内容の詳細

### 2.1 MultilingualPhonemizerOptions クラスの新規作成

**ファイル**: `Assets/uPiper/Runtime/Core/Phonemizers/Multilingual/MultilingualPhonemizerOptions.cs`（新規作成）

```csharp
using System.Collections.Generic;
using DotNetG2P.Chinese;
using DotNetG2P.English;
using DotNetG2P.French;
using DotNetG2P.Korean;
using DotNetG2P.Portuguese;
using DotNetG2P.Spanish;
using uPiper.Core.Phonemizers.Backend;
using uPiper.Core.Phonemizers.Implementations;

namespace uPiper.Core.Phonemizers.Multilingual
{
    /// <summary>
    /// Configuration options for MultilingualPhonemizer.
    /// Groups constructor parameters into a single configuration object.
    /// </summary>
    public class MultilingualPhonemizerOptions
    {
        /// <summary>Languages to support (e.g., ["ja", "en"]). Required, must not be null or empty.</summary>
        public IReadOnlyList<string> Languages { get; set; }

        /// <summary>Default language for Latin text (default: "en").</summary>
        public string DefaultLatinLanguage { get; set; } = "en";

        /// <summary>Optional pre-built Japanese phonemizer; one is created if null.</summary>
        public DotNetG2PPhonemizer JaPhonemizer { get; set; }

        /// <summary>Optional pre-built English G2P engine (DotNetG2P.English).</summary>
        public EnglishG2PEngine EnEngine { get; set; }

        /// <summary>Optional pre-built Spanish G2P engine (DotNetG2P.Spanish).</summary>
        public SpanishG2PEngine EsEngine { get; set; }

        /// <summary>Optional pre-built French G2P engine (DotNetG2P.French).</summary>
        public FrenchG2PEngine FrEngine { get; set; }

        /// <summary>Optional pre-built Portuguese G2P engine (DotNetG2P.Portuguese).</summary>
        public PortugueseG2PEngine PtEngine { get; set; }

        /// <summary>Optional pre-built Chinese G2P engine (DotNetG2P.Chinese).</summary>
        public ChineseG2PEngine ZhEngine { get; set; }

        /// <summary>Optional pre-built Korean G2P engine (DotNetG2P.Korean).</summary>
        public KoreanG2PEngine KoG2PEngine { get; set; }

        /// <summary>
        /// Optional pre-built English phonemizer backend (legacy, for test stub injection).
        /// Prefer EnEngine for production use.
        /// </summary>
        [System.Obsolete("Use EnEngine instead. Legacy backend kept for backward compatibility with test stubs.")]
        public IPhonemizerBackend EnPhonemizer { get; set; }

        /// <summary>
        /// Optional pre-built Korean phonemizer backend (legacy).
        /// Prefer KoG2PEngine for production use.
        /// </summary>
        [System.Obsolete("Use KoG2PEngine instead. Legacy backend kept for backward compatibility.")]
        public IPhonemizerBackend KoPhonemizer { get; set; }

        /// <summary>
        /// Validates the options. Throws ArgumentException if invalid.
        /// </summary>
        public void Validate()
        {
            if (Languages == null || Languages.Count == 0)
                throw new System.ArgumentException("At least one language must be specified.", nameof(Languages));

            if (string.IsNullOrWhiteSpace(DefaultLatinLanguage))
                DefaultLatinLanguage = "en";
        }
    }
}
```

### 2.2 MultilingualPhonemizer に新コンストラクタを追加

**ファイル**: `Assets/uPiper/Runtime/Core/Phonemizers/Multilingual/MultilingualPhonemizer.cs`

**(a) 新コンストラクタの追加（L96付近に挿入）**:

```csharp
/// <summary>
/// Creates a MultilingualPhonemizer from an options object.
/// </summary>
/// <param name="options">Configuration options.</param>
public MultilingualPhonemizer(MultilingualPhonemizerOptions options)
{
    if (options == null)
        throw new ArgumentNullException(nameof(options));
    options.Validate();

    _languages = options.Languages;
    _languageSet = new HashSet<string>(options.Languages);
    _defaultLatinLanguage = options.DefaultLatinLanguage;
    _detector = new UnicodeLanguageDetector(options.Languages, options.DefaultLatinLanguage);
    _jaPhonemizer = options.JaPhonemizer;
#pragma warning disable CS0618 // Obsolete
    _enPhonemizer = options.EnPhonemizer;
#pragma warning restore CS0618
    _esEngine = options.EsEngine;
    _frEngine = options.FrEngine;
    _ptEngine = options.PtEngine;
    _zhEngine = options.ZhEngine;
#pragma warning disable CS0618 // Obsolete
    _koPhonemizer = options.KoPhonemizer;
#pragma warning restore CS0618
    _koG2PEngine = options.KoG2PEngine;
}
```

**(b) 旧コンストラクタに [Obsolete] を付与（L96-106）**:

```csharp
/// <summary>
/// Creates a MultilingualPhonemizer for the specified language list.
/// </summary>
[Obsolete("Use the constructor that takes MultilingualPhonemizerOptions instead. This constructor will be removed in v2.0.")]
public MultilingualPhonemizer(
    IReadOnlyList<string> languages,
    string defaultLatinLanguage = "en",
    DotNetG2PPhonemizer jaPhonemizer = null,
    IPhonemizerBackend enPhonemizer = null,
    SpanishG2PEngine esEngine = null,
    FrenchG2PEngine frEngine = null,
    PortugueseG2PEngine ptEngine = null,
    ChineseG2PEngine zhEngine = null,
    IPhonemizerBackend koPhonemizer = null,
    KoreanG2PEngine koG2PEngine = null)
```

旧コンストラクタの本体コードは変更なし。

**(c) EnglishG2PEngine フィールドの対応**:

現在 `_enEngine` フィールド（L59）は `InitializeAsync` 内でのみ生成される。Options から渡す場合のために、新コンストラクタで `_enEngine = options.EnEngine;` を追加する。ただし `_enEngine` への外部注入パスは InitializeAsync 内の `_enEngine == null && _enPhonemizer == null` チェック（L149）で制御されるため、Options.EnEngine が非null の場合は InitializeAsync での自動生成がスキップされる。

## 5. 懸念事項・レビュー項目

### 5.1 韓国語のバックエンド共存

`MultilingualPhonemizer` には韓国語用に `_koPhonemizer`（レガシー `IPhonemizerBackend`）と `_koG2PEngine`（`KoreanG2PEngine`）が共存している。Options クラスでは両方のプロパティを保持するが、`KoPhonemizer` には `[Obsolete]` を付与し、将来的に `KoG2PEngine` のみに統一する方針を明示する。

`PhonemizeWithProsodyAsync` 内の分岐（L468）では `_koG2PEngine != null` が優先されるため、両方が設定された場合は `KoG2PEngine` が使用される。この動作は既存と同一。

### 5.2 EnglishG2PEngine の注入パス

現在 `_enEngine` は `InitializeAsync` 内でのみ生成される（L149-180）。Options に `EnEngine` プロパティを追加することで、テスト時に事前構築済みの `EnglishG2PEngine` を注入可能になる。ただし `_enEngine` がコンストラクタで設定された場合、`_ownsEn` フラグは `false` のままであり、Dispose 時に外部注入されたエンジンは解放されない（所有権は呼び出し元に残る）。この動作は `_jaPhonemizer` の既存パターンと一致する。

### 5.3 テストファイルの `[Obsolete]` 警告

旧コンストラクタを使用する既存テスト約20箇所でコンパイル時に CS0618 警告が発生する。`TreatWarningsAsErrors` が有効な場合はビルドエラーになる可能性がある。

**緩和策**: テストファイルの先頭に `#pragma warning disable CS0618` を追加する。本チケットのスコープで各テストファイルに追加する。
