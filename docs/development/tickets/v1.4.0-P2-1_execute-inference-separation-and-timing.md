# v1.4.0-P2-1: ExecuteInference 3段階分離 + 推論タイミングログ

## 1. タスク目的とゴール

`InferenceAudioGenerator.ExecuteInference`（L420-512）は、テンソル構築・推論実行・結果取得の3つの責務を1メソッド内に集約しており、個別のプロファイリングや最適化が困難になっている。本チケットでは以下を実施する:

1. ExecuteInferenceを3つのprivateメソッド（PrepareInputs / RunInference / ExtractResults）に分離する
2. 各段階にStopwatchベースのタイミング計測を追加し、推論パフォーマンスの可視化を実現する
3. ExecuteWarmupを最適化し、不要なReadbackAndClone呼び出しを回避する

**ゴール**:
- ExecuteInferenceが3つの明確な段階に分離され、各段階の責務が単一になっている
- 推論タイミングが内訳付きでログ出力され、パフォーマンスボトルネックの特定が容易になる
- ExecuteWarmupがPrepareInputs+RunInferenceのみを実行し、GPU→CPUリードバックのオーバーヘッドを回避する
- 既存の外部インターフェースに変更がなく、全テストが通過する

**(d) ExecuteInferenceファサード（L420-512）**:

3段階を順次呼び出し、既存のtry-finally構造を維持する。

```csharp
private float[] ExecuteInference(
    int[] phonemeIds,
    int[] prosodyA1, int[] prosodyA2, int[] prosodyA3,
    float lengthScale, float noiseScale, float noiseW,
    int speakerId = 0, int languageId = 0)
{
    var hasAnyProsodyInput = prosodyA1 != null || prosodyA2 != null || prosodyA3 != null;
    // --- タイミング計測（2.2節参照） ---

    (Tensor<int> input, Tensor<int> inputLengths, Tensor<float> scales,
     Tensor<int> prosody, int[] rentedProsody, Tensor<int> sid, Tensor<int> lid) inputs = default;
    Tensor<float> outputTensor = null;
    Tensor<float> readableTensor = null;

    try
    {
        inputs = PrepareInputs(phonemeIds, prosodyA1, prosodyA2, prosodyA3,
                               lengthScale, noiseScale, noiseW, speakerId, languageId);

        RunInference();

        var (audioData, outTensor, readTensor) = ExtractResults();
        outputTensor = outTensor;
        readableTensor = readTensor;

        return audioData;
    }
    catch (Exception ex)
    {
        PiperLogger.LogError($"[InferenceAudioGenerator] Failed to execute inference: {ex.Message}");
        throw;
    }
    finally
    {
        readableTensor?.Dispose();
        outputTensor?.Dispose();
        inputs.input?.Dispose();
        inputs.inputLengths?.Dispose();
        inputs.scales?.Dispose();
        inputs.sid?.Dispose();
        inputs.lid?.Dispose();
        inputs.prosody?.Dispose();
        if (inputs.rentedProsody != null)
            ArrayPool<int>.Shared.Return(inputs.rentedProsody);
    }
}
```

### 2.2 タイミングログの追加

**ファイル**: `Assets/uPiper/Runtime/Core/AudioGeneration/InferenceAudioGenerator.cs`

> `System.Diagnostics.Stopwatch` の使用に必要。既存の `using System;` の後に追加する。

**(b) ExecuteInferenceファサード内でのStopwatch計測**:

各段階の前後で `Stopwatch` を使用して計測する。

```csharp
var sw = Stopwatch.StartNew();

// PrepareInputs
inputs = PrepareInputs(...);
var prepareMs = sw.ElapsedMilliseconds;

// RunInference
sw.Restart();
RunInference();
var scheduleMs = sw.ElapsedMilliseconds;

// ExtractResults
sw.Restart();
var (audioData, outTensor, readTensor) = ExtractResults();
var extractMs = sw.ElapsedMilliseconds;

var totalMs = prepareMs + scheduleMs + extractMs;
PiperLogger.LogInfo(
    $"[InferenceAudioGenerator] Inference took {totalMs}ms " +
    $"(prepare: {prepareMs}ms, schedule: {scheduleMs}ms, readback: {extractMs}ms)");
```

- `PiperLogger.LogInfo` を使用し、既存のログパターン `[InferenceAudioGenerator]` プレフィックスに合わせる
- 既存のL432-433、L478-480のログ出力はタイミングログに統合して削除する

## 5. 懸念事項・レビュー項目

### 5.1 テンソル所有権管理

PrepareInputsが作成したテンソルの所有権はファサード（ExecuteInference）のfinally節が管理する。3メソッドに分離しても、テンソルのライフサイクルはファサード内に閉じているため、リークリスクは低い。

**レビューポイント**: PrepareInputsの戻り値タプルに含まれる全テンソルがfinally節でDispose対象になっているか確認すること。

### 5.2 ArrayPool返却タイミング

`CreateProsodyTensorPooled`（L536-563）で借りた配列は、Sentisがテンソル作成時にデータをコピーするため、テンソル作成直後に返却可能。ただし、現行コード（L509-510）ではfinally節で返却しており、この安全なパターンを維持する。

### 5.3 P1-1との依存関係

P1-1（ExtractAudioData→ToArray()置換）が完了していない場合、ExtractResults内で `ExtractAudioData(readableTensor)` をそのまま使用する。P1-1完了後は `readableTensor.ToArray()` に置き換わるため、ExtractResultsの実装は軽微に変更される。

**推奨**: P1-1を先に完了させてからP2-1に着手する。ただし、P1-1未完了でもP2-1の実装は可能（ExtractAudioDataを使用するだけ）。

### 5.4 Stopwatchの精度

`Stopwatch.ElapsedMilliseconds` はlong型（ミリ秒単位）。GPUスケジューリングが非常に高速（1ms未満）の場合、schedule=0msと表示される可能性がある。必要に応じて `sw.Elapsed.TotalMilliseconds` (double型) への変更を検討するが、初期実装ではミリ秒単位で十分。

### 5.5 レビュー重点項目

- [ ] PrepareInputsの戻り値タプルがfinally節の全Disposeと一致しているか
- [ ] RunInferenceが副作用（テンソル作成等）を持たず、Schedule()のみであるか
- [ ] ExtractResultsが返すテンソルがファサードのfinally節で正しくDisposeされるか
- [ ] ExecuteWarmupがExtractResults/ReadbackAndCloneを呼ばないことが明確か
- [ ] タイミングログのフォーマットが既存の `[InferenceAudioGenerator]` パターンと一貫しているか
- [ ] `using System.Diagnostics` がUnity環境で問題なくコンパイルできるか（標準.NETなので問題なし）
