## 1. タスク目的とゴール

`PiperTTS.Inference.cs` の3分岐パス（通常/Prosody/Silence分割）に対するモックベース統合テストを追加する。TTSSynthesisOrchestrator と SplitInferenceOrchestrator の各パスが正しく動作することを、スタブを用いて検証する。

**ゴール**:
- TTSSynthesisOrchestrator の3パス（通常/Prosody/Silence分割）がテストでカバーされている
- SplitInferenceOrchestrator の基本動作（単一句/複数句/空入力）がテストでカバーされている
- テスト用スタブ（StubInferenceAudioGenerator）が再利用可能なテストインフラとして存在する
- 既存テストが変更なしでパス

## 2. 実装する内容の詳細

### 2.1 テスト対象

#### TTSSynthesisOrchestrator

音素列からAudioClip変換パイプラインの3パスをテストする:

| パス | 条件 | 検証ポイント |
|------|------|------------|
| 通常パス | Prosodyなし、Silence分割なし | 基本的な音素→音声変換が正しく動作する |
| Prosodyパス | Prosodyあり | Prosody情報（A1/A2/A3）が正しく推論に渡される |
| Silence分割パス | EnablePhonemeSilence=true | 沈黙句分割が有効な場合の動作が正しい |

#### SplitInferenceOrchestrator

沈黙句分割の正しい分割・結合動作をテストする:

| テストケース | 内容 | 検証ポイント |
|------------|------|------------|
| 単一句 | 分割なし | 分割不要な入力が正しく処理される |
| 複数句 | 無音挿入 | 句間に無音が正しく挿入される |
| 空入力 | 空の音素列 | 空入力に対して例外が発生しない |

### 2.2 テストインフラ

#### StubInferenceAudioGenerator

`IInferenceAudioGenerator` のテスト用スタブ:

- 固定audio data（float配列）を返す
- 呼び出し回数・引数を記録（テスト側で検証可能）
- Prosody対応/非対応を切り替え可能
- パラメータ記録: LastLengthScale, LastNoiseScale, LastNoiseW, LastSpeakerId, LastLanguageId

### 2.3 テスト構成

**ファイル配置**: `Assets/uPiper/Tests/Editor/AudioGeneration/` 配下

| ファイル | テスト数 | 内容 |
|---------|---------|------|
| `TTSSynthesisOrchestratorTests.cs` | 12 | 3パスの統合テスト + パラメータ伝播・エンコード結果検証・config=null |
| `SplitInferenceOrchestratorTests.cs` | 5 | 分割・結合動作テスト + Prosody付き句分割 |
| `StubInferenceAudioGenerator.cs` | - | 共有テストスタブ |

## 4. 提供範囲・テスト項目

### 提供範囲 (Scope)

**In-scope**:
- TTSSynthesisOrchestratorTests（12テスト）
- SplitInferenceOrchestratorTests（5テスト）
- StubInferenceAudioGenerator（共有スタブ、パラメータ記録対応）

**Out-of-scope**:
- PiperTTSレベルのEnd-to-End統合テスト（モデルアセット依存）
- MultilingualPhonemizerの統合テスト（既存テストでカバー）

### テスト項目

| テストケース | 内容 | 検証ポイント |
|------------|------|------------|
| 通常パス基本動作 | Prosodyなし・Silence分割なしで音声生成 | AudioClipが正しく生成される |
| Prosodyパス基本動作 | Prosody情報付きで音声生成 | A1/A2/A3が推論に渡される |
| Silence分割パス基本動作 | EnablePhonemeSilence=trueで音声生成 | SplitInferenceOrchestratorが呼ばれる |
| パラメータ伝播 | SynthesisRequest経由でパラメータを渡す | LengthScale/NoiseScale/NoiseW/SpeakerId/LanguageIdがスタブに正しく伝播される |
| エンコード結果検証 | PhonemeEncoderの出力を検証 | エンコード後のphonemeIdsが期待通りである |
| config=null処理 | ValidatedPiperConfig=nullで呼び出し | 例外なしでデフォルト動作する |
| 単一句分割 | 分割不要な入力 | 分割なしで正しく処理される |
| 複数句分割 | 複数句の入力 | 句間に無音が挿入される |
| 空入力処理 | 空の音素列 | 例外なしで空結果を返す |
| Prosody付き句分割 | Prosody情報を含む複数句の入力 | Prosody配列が句ごとに正しく分割・推論に渡される |
| スタブ引数記録 | StubInferenceAudioGeneratorの引数検証 | 渡された引数が正しく記録される |
