# Android パフォーマンス最適化計画

## 概要

Phase 2完了後のAndroidパフォーマンス最適化を実施します。現在の基本実装は動作していますが、モバイルデバイス向けに更なる最適化が必要です。

## 現状分析

### 現在のパフォーマンス指標
- **音素化ライブラリ**: dot-net-g2p（純粋C#実装、ネイティブライブラリ不要）
- **辞書データ**: MeCab辞書（NAIST Japanese Dictionary）展開後約50MB
- **初期化時間**: 初回起動時は辞書展開のため数秒かかる
- **メモリ使用量**: 辞書ロード後約50-70MB追加

### ボトルネック
1. **辞書データのサイズと展開時間**
   - 50MBの辞書データ展開に時間がかかる
   - メモリに全て保持している

2. **文字列処理のオーバーヘッド**
   - UTF-8/UTF-16変換が頻繁
   - 辞書検索時のメモリ割り当て

## 最適化戦略

### 1. 辞書データの最適化（優先度: 高）

#### 1.1 辞書データの圧縮
- **手法**: 
  - ZIP圧縮での配布（StreamingAssets）
  - 展開時の段階的処理
  - 必要な部分のみメモリロード
- **期待効果**: 
  - APKサイズ20-30MB削減
  - 初期化時間の短縮

#### 1.2 辞書の遅延ロード
- **手法**:
  - 必要な辞書エントリのみロード
  - LRUキャッシュの実装
  - メモリマップファイルの検討
- **期待効果**:
  - メモリ使用量50%削減
  - 初期化時間を1秒以内に

### 2. dot-net-g2p最適化（優先度: 中）

> **注**: 現在はdot-net-g2p（純粋C#実装）を使用しているため、ネイティブライブラリのビルド最適化やP/Invoke最適化は不要です。

#### 2.1 辞書検索の最適化
- MeCab辞書のメモリ内インデックス最適化
- 頻出語のキャッシュ

#### 2.2 文字列処理の最適化
- Span<T>を使用したアロケーション削減
- 文字列バッファの再利用

### 3. Unity側の最適化（優先度: 高）

#### 4.1 非同期処理の改善
```csharp
public class OptimizedAndroidPathResolver
{
    private static Task<bool> _dictionaryExtractionTask;
    
    // アプリ起動時に非同期で辞書展開を開始
    public static void PreloadDictionaryAsync()
    {
        if (_dictionaryExtractionTask == null)
        {
            _dictionaryExtractionTask = Task.Run(() => ExtractDictionaryFiles());
        }
    }
    
    // 実際に使用する時に完了を待つ
    public static async Task<string> GetDictionaryPathAsync()
    {
        if (_dictionaryExtractionTask != null)
        {
            await _dictionaryExtractionTask;
        }
        return GetDictionaryPath();
    }
}
```

#### 4.2 キャッシュの強化
- 音素化結果のキャッシュサイズ最適化
- WeakReferenceを使用したメモリ効率化

### 4. IL2CPP最適化（優先度: 低）

#### 4.1 dot-net-g2pのIL2CPP互換性
- リフレクション使用箇所のlink.xml対応

## 実装結果

### アーキテクチャ移行の成果

dot-net-g2p（純粋C#実装）への移行により、ネイティブライブラリが不要になりました：

| 項目 | 旧実装（OpenJTalk） | 現実装（dot-net-g2p） |
|------|---------------------|----------------------|
| ネイティブライブラリ | 全ABI合計約20MB | 不要（0MB） |
| ビルド複雑度 | NDK + CMake + Docker | NuGetパッケージ参照のみ |
| クロスプラットフォーム | ABI別ビルド必要 | 純粋C#で自動対応 |

### 実装された最適化

1. **純粋C#移行**
   - ネイティブライブラリ（.so）不要
   - P/Invoke境界のオーバーヘッド解消
   - IL2CPP互換性の簡素化

2. **コード最適化**
   - 非同期辞書展開
   - 遅延ロード実装

3. **リソース最適化**
   - 辞書データZIP圧縮（予定）

## リスクと対策

### リスク1: 辞書圧縮による処理時間増加
- **対策**: 事前展開とキャッシュの組み合わせ
- **代替案**: 部分的な圧縮（頻出語のみ非圧縮）

### リスク2: メモリ不足によるクラッシュ
- **対策**: OutOfMemoryの適切なハンドリング
- **代替案**: 品質を落とした軽量辞書の提供

### リスク3: 古いデバイスでの互換性問題
- **対策**: Android 5.0以上を最小要件に
- **代替案**: レガシーモードの実装
