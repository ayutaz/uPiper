# P3-1: SplitInferenceOrchestrator durations 結合

## 1. タスク目的とゴール

`SplitInferenceOrchestrator` の戻り値を `Task<NativeArray<float>>` から `Task<InferenceOutput>` に変更し、句ごとに分割推論された durations を結合して返却する。これにより、沈黙句分割を経由した場合でも下流の `TTSSynthesisOrchestrator`（P3-2）が正しい durations を受け取り、`TimingCalculator` でタイミング情報を算出できるようになる。

### なぜ必要か

- P2-1 で `IInferenceAudioGenerator.GenerateAudioAsync` の戻り値が `Task<InferenceOutput>` に変更されたが、`SplitInferenceOrchestrator` は P2-1 時点では暫定的に `InferenceOutput` から Audio のみ抽出し、Durations を破棄する実装になっている
- 沈黙句分割を使用する場合、句ごとの推論結果に含まれる durations を正しく結合しなければ、タイミング情報が失われる
- TTSSynthesisOrchestrator（P3-2）は SplitInferenceOrchestrator の戻り値から durations を取得して TimingCalculator に渡す設計のため、ここで durations を伝搬する必要がある
- 句間無音（SilenceSamples）は durations に含めない。タイミング計算時のオフセット加算は TTSSynthesisOrchestrator 側の責務とする

### 2.3 SplitInferenceOrchestrator.cs の変更

メソッドシグネチャと内部ロジックを変更する。主要な変更点は以下の3つ:

1. **戻り値型変更**: `Task<NativeArray<float>>` から `Task<InferenceOutput>` へ
2. **durations 累積結合**: 句ごとの `InferenceOutput.Durations` を `List<float>` に追加
3. **InferenceOutput 構築**: 結合した Audio と Durations を `InferenceOutput` にまとめて返却

```csharp
/// <remarks>Caller owns and must Dispose the returned InferenceOutput.</remarks>
public async Task<InferenceOutput> GenerateWithSilenceSplitAsync(
    int[] phonemeIds,
    int[] prosodyFlat,
    IReadOnlyDictionary<string, float> phonemeSilence,
    IReadOnlyDictionary<string, int[]> phonemeIdMap,
    int sampleRate,
    float lengthScale = 1.0f,
    float noiseScale = 0.667f,
    float noiseW = 0.8f,
    int speakerId = 0,
    int languageId = 0,
    IProgress<float> progress = null,
    CancellationToken cancellationToken = default)
{
    var phrases = PhonemeSilenceProcessor.SplitAtPhonemeSilence(
        phonemeIds, prosodyFlat,
        phonemeSilence, phonemeIdMap, sampleRate);

    PiperLogger.LogInfo(
        $"[SplitInferenceOrchestrator] Silence split: {phrases.Count} phrases " +
        $"from {phonemeIds.Length} phonemes");

    if (phrases.Count == 0)
    {
        PiperLogger.LogWarning(
            "[SplitInferenceOrchestrator] No phrases after silence split");
        return new InferenceOutput(
            new NativeArray<float>(0, Allocator.Persistent),
            default);
    }

    // Count non-empty phrases for accurate progress reporting
    var totalPhrases = 0;
    for (var i = 0; i < phrases.Count; i++)
    {
        var ph = phrases[i];
        if (ph.PhonemeIds != null && ph.PhonemeIds.Length > 0)
            totalPhrases++;
    }

    var segments = new List<(NativeArray<float> Audio, int SilenceSamples)>();
    var allDurations = new List<float>();
    var hasDurations = false;
    NativeArray<float> combinedAudio = default;
    NativeArray<float> combinedDurations = default;

    try
    {
        var totalLength = 0;
        var completedPhrases = 0;

        for (var p = 0; p < phrases.Count; p++)
        {
            var phrase = phrases[p];
            if (phrase.PhonemeIds == null || phrase.PhonemeIds.Length == 0)
                continue;

            cancellationToken.ThrowIfCancellationRequested();

            var phraseOutput = await _generator.GenerateAudioAsync(
                phrase.PhonemeIds, phrase.ProsodyFlat,
                lengthScale, noiseScale, noiseW,
                speakerId, languageId,
                cancellationToken);

            var phraseDurationsCount = 0;
            try
            {
                // Audio: 句ごとの NativeArray をコピーして保持
                var phraseAudio = new NativeArray<float>(
                    phraseOutput.Audio.Length, Allocator.Persistent);
                NativeArray<float>.Copy(phraseOutput.Audio, phraseAudio);
                segments.Add((phraseAudio, phrase.SilenceSamples));
                totalLength += phraseAudio.Length + phrase.SilenceSamples;

                // Durations: 句ごとのdurationsをList<float>に累積
                // 句間無音はdurationsに含めない
                if (phraseOutput.HasDurations)
                {
                    hasDurations = true;
                    phraseDurationsCount = phraseOutput.Durations.Length;
                    for (var d = 0; d < phraseDurationsCount; d++)
                    {
                        allDurations.Add(phraseOutput.Durations[d]);
                    }
                }
            }
            finally
            {
                // 句ごとの InferenceOutput を確実に Dispose
                phraseOutput.Dispose();
            }

            completedPhrases++;

            // Report per-phrase progress (0.0 to 1.0)
            if (totalPhrases > 0)
            {
                progress?.Report((float)completedPhrases / totalPhrases);
            }

            PiperLogger.LogDebug(
                $"[SplitInferenceOrchestrator] Phrase {completedPhrases}/{totalPhrases}: " +
                $"{segments[segments.Count - 1].Audio.Length} samples + " +
                $"{phrase.SilenceSamples} silence" +
                (hasDurations
                    ? $", {phraseDurationsCount} durations"
                    : ""));
        }

        // --- Audio 結合 ---
        // ClearMemory ensures silence gaps are zero-initialized
        combinedAudio = new NativeArray<float>(
            totalLength, Allocator.Persistent, NativeArrayOptions.ClearMemory);
        var offset = 0;
        foreach (var (audio, silenceSamples) in segments)
        {
            NativeArray<float>.Copy(audio, 0, combinedAudio, offset, audio.Length);
            offset += audio.Length;
            offset += silenceSamples; // Zero-initialized (silence)
        }

        // --- Durations 結合 ---
        if (hasDurations && allDurations.Count > 0)
        {
            combinedDurations = new NativeArray<float>(
                allDurations.Count, Allocator.Persistent);
            for (var i = 0; i < allDurations.Count; i++)
            {
                combinedDurations[i] = allDurations[i];
            }
        }

        PiperLogger.LogInfo(
            $"[SplitInferenceOrchestrator] Silence split complete: " +
            $"{totalLength} total samples ({segments.Count} phrases)" +
            (hasDurations
                ? $", {allDurations.Count} total durations"
                : ", no durations"));

        return new InferenceOutput(combinedAudio, combinedDurations);
    }
    catch
    {
        if (combinedAudio.IsCreated)
            combinedAudio.Dispose();
        if (combinedDurations.IsCreated)
            combinedDurations.Dispose();
        throw;
    }
    finally
    {
        // Dispose per-phrase Audio NativeArrays
        // (phraseOutput は try/finally で既に Dispose 済み)
        foreach (var (audio, _) in segments)
        {
            if (audio.IsCreated)
                audio.Dispose();
        }
    }
}
```

### 2.5 句間無音と durations の関係

句間無音（`SilenceSamples`）は以下の理由で durations に含めない:

1. **durations はモデル出力**: durations テンソルは ONNX モデルが各音素に対して出力するフレーム持続時間であり、句間に挿入する人工的な無音とは無関係
2. **TimingCalculator の責務分離**: `TimingCalculator.Calculate()` は単一推論結果の durations を処理する。句間無音のオフセット加算は上位層（TTSSynthesisOrchestrator、P3-2）の責務
3. **結合順序**: Audio 結合時は `[phrase1_audio][silence][phrase2_audio]` だが、Durations 結合は `[phrase1_durations][phrase2_durations]` となる。無音部分に対応する durations エントリは存在しない

TTSSynthesisOrchestrator（P3-2）では、句分割を使用した場合に `TimingCalculator` の結果に句間無音のオフセットを加算するロジックが必要になる。ただし、P3-2 の設計判断として「句分割時はタイミング情報を提供しない（HasTimings=false）」とする選択肢もあり得る。この判断は P3-2 に委ねる。

### 2.6 Durations 結合の正当性

各句は独立した推論として実行され、それぞれの durations は句内の音素に対するフレーム持続時間を表す。句ごとの durations を単純結合した配列は、元の（分割前の）phonemeIds に対応する durations 配列と等価になる。

```
元の phonemeIds: [BOS, p1, p2, silence_token, p3, p4, EOS]
分割後:
  句1: [BOS, p1, p2, EOS]  → durations1: [d_BOS1, d_p1, d_p2, d_EOS1]
  句2: [BOS, p3, p4, EOS]  → durations2: [d_BOS2, d_p3, d_p4, d_EOS2]

結合durations: [d_BOS1, d_p1, d_p2, d_EOS1, d_BOS2, d_p3, d_p4, d_EOS2]
```

注意: 分割後の各句には独自の BOS/EOS が付与される。結合 durations にはこれらが含まれる。TimingCalculator は BOS/EOS を自動スキップするため、タイミング計算に影響はない。

### 2.7 hasDurations フラグの設計

`hasDurations` フラグは「いずれかの句が durations を持っていた」ことを示す。全句が durations を持たない場合（`hasDurations == false`）、結合 durations は `default(NativeArray<float>)` となる。

一部の句のみ durations を持つ不整合な状態は想定しない。`SupportsDurations` はモデル単位のプロパティであり、同一推論セッション内で変化しないため、全句が同一の durations サポート状態を持つ。

### 2.8 ログメッセージ

P3-1 ではログメッセージに durations 関連の情報を追加する:

- 句ごとのログ: `"Phrase N/M: X samples + Y silence, Z durations"`
- 完了ログ: `"Silence split complete: X total samples (N phrases), Y total durations"`

### 2.9 Audio コピーのオーバーヘッド

P2-1 の暫定実装と同様、句ごとの Audio を新しい NativeArray にコピーする。`phraseOutput.Dispose()` で元の Audio が破棄されるため、コピーが必要。

将来の最適化として `phraseOutput` から Audio の所有権を移転（move semantics）する方法が考えられるが、`InferenceOutput` は `readonly struct` のため内部フィールドの変更ができない。Audio コピーサイズは通常 100KB-1MB（22050Hz x 0.5-5秒/句）であり、推論時間（数十ms-数百ms/句）と比較して無視できるため、Phase 3 ではコピーで対応する。

### 2.10 空フレーズ時の挙動

`phrases.Count == 0` の場合、空の Audio（長さ0の NativeArray）と `default` の Durations を持つ `InferenceOutput` を返す。これは P2-1 暫定実装の空の NativeArray 返却と整合する。

### 5.1 懸念事項

| # | 懸念 | 対策 |
|---|------|------|
| 1 | `List<float>` への逐次追加による GC アロケーション | durations の要素数は通常 40-200 程度（音素数と同じ）。List の初期容量を `phonemeIds.Length` にすれば realloc は発生しない。ただし句分割後の各句の phonemeIds 長は事前に不明なため、デフォルト容量（空 List）で開始する。要素数が少ないため GC 圧力は無視できる |

## 6. 一から作り直すとしたら

### 代替案A: InferenceOutput の Audio 所有権移転

```csharp
// phraseOutput から Audio を「移動」し、Durations のみ Dispose
var phraseAudio = phraseOutput.TakeAudio(); // Audio の所有権を移転
phraseOutput.DisposeDurationsOnly(); // Durations のみ Dispose
```

**メリット**: Audio のコピーが不要。メモリ効率が向上。
**デメリット**: `InferenceOutput` は `readonly struct` のため、内部フィールドを変更する「move」操作が言語仕様上できない。`ref` パラメータや `[UnscopedRef]` 等の低レベル手法が必要で複雑化。

**判定**: Audio コピーコストは推論時間と比較して無視できるため、シンプルなコピーで対応する。将来パフォーマンスが問題になった場合に検討。

### 代替案B: 句分割時は durations を返さない

```csharp
// 句分割使用時は常に HasDurations = false
return new InferenceOutput(combinedAudio, default);
```

**メリット**: 実装が大幅に簡略化。句間無音のオフセット問題を回避。
**デメリット**: 句分割使用時にタイミング情報が一切利用不可。句分割は長文テキストで有効化されるため、まさにタイミング情報が最も有用なケースで利用不可になる。

**判定**: タイミング情報の有用性を優先し、durations 結合を実装する。句間無音のオフセット問題は P3-2 で対処する。

### 代替案C: NativeArray の結合に NativeList を使用

```csharp
// NativeList<float> で durations を逐次追加し、最後に AsArray() で変換
var durList = new NativeList<float>(phonemeIds.Length, Allocator.Persistent);
// ...
durList.AddRange(phraseOutput.Durations);
// ...
var combinedDurations = durList.AsArray();
```

**メリット**: List<float>（managed）を介さず、全て unmanaged メモリで処理。GC アロケーションなし。
**デメリット**: `NativeList` は `Unity.Collections` の追加型であり、Dispose 管理が複雑化。`AsArray()` は NativeList の内部バッファの view を返すため、NativeList を Dispose すると view も無効になる。安全なコピーには結局 `NativeArray` への明示的コピーが必要。

**判定**: durations は通常 200 要素未満であり、`List<float>` の GC 圧力は Gen0 で即回収。コード簡潔性を優先し `List<float>` を使用する。

## 7. 後続タスクへの連絡事項

### P3-2 (TTSSynthesisOrchestrator タイミング計算統合) への連絡

- `ISplitInferenceOrchestrator.GenerateWithSilenceSplitAsync` の戻り値が `InferenceOutput` に変更済み
- `InferenceOutput.Audio` は句ごとの Audio を結合した NativeArray（句間に SilenceSamples 分のゼロパディングを含む）
- `InferenceOutput.Durations` は句ごとの durations を単純結合した NativeArray（句間無音に対応するエントリは含まない）
- 句分割使用時の durations は「句1の全durations + 句2の全durations + ...」の形式。各句の BOS/EOS durations を含む
- TTSSynthesisOrchestrator で句分割を使用した場合のタイミング計算は以下のいずれかで対処すること:
  - 方式A: 結合 durations を TimingCalculator に渡し、句間無音のオフセットは PhonemeTimingEntry の StartSeconds/EndSeconds に反映しない（句内相対時間のまま）
  - 方式B: 句分割使用時は `HasTimings = false` として SynthesisWithTimingResult を構築（シンプルだがタイミング情報が失われる）
  - 方式C: 句ごとに TimingCalculator を呼び出し、句間無音の累積オフセットを加算して結果をマージ
  - **推奨**: P3-2 担当者が判断。方式C が最も正確だが実装コストが高い

### T3-1 (SplitInferenceOrchestrator 句分割タイミングテスト) への連絡

- テストには `StubInferenceAudioGenerator`（P2-4）を使用
- `DurationsToReturn` に句ごとの既知の durations を設定
- 検証ポイント:
  - 戻り値の `HasDurations == true`（StubInferenceAudioGenerator が durations を返す設定の場合）
  - `Durations.Length` == 全句の durations 合計長
  - `Durations` の値が句ごとの durations を順番に結合した配列と一致
  - `Audio.Length` == 全句の Audio 合計長 + 全句間の SilenceSamples 合計
- NativeArray のテストでは `try/finally` で Dispose 漏れを防止すること

### P3-3 (AudioSynthesisCache タイミング格納拡張) への連絡

- SplitInferenceOrchestrator からの durations は TTSSynthesisOrchestrator（P3-2）経由で TimingCalculator に渡され、PhonemeTimingEntry[] に変換される
- AudioSynthesisCache に格納されるのは PhonemeTimingEntry[]（P3-3 のスコープ）であり、生の durations 配列ではない
