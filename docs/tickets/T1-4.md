# T1-4: エッジケーステスト

## 1. タスク目的とゴール

TimingCalculator.Calculate() のエッジケースを網羅的にテストし、異常入力・境界値に対して例外・クランプ・安全なフォールバックが正しく動作することを検証する。要件定義 E2-E8 のうち TimingCalculator 責務のケースをカバーする。

## 2. 実装する内容の詳細

### テストファイル
`Assets/uPiper/Tests/Editor/AudioGeneration/TimingCalculatorTests.cs`（T1-1 で作成済みのファイルに追加）

### 対応するエッジケース要件

| 要件 | ケース | 期待動作 | テストメソッド |
|------|--------|---------|--------------|
| E2 | durations.Length != phonemeIds.Length | Math.Min で打ち切り | Calculate_LengthMismatch_UsesMinLength |
| E3 | durations 全ゼロ | 全エントリ DurationSeconds == 0 | Calculate_AllZeroDurations_NoCrash |
| E4 | durations に負値 | Math.Max(0, frameDuration) でクランプ | Calculate_NegativeDurations_ClampsToZero |
| E6 | 空テキスト（音素0個） | 空リスト返却 | Calculate_EmptyInput_ReturnsEmpty |
| E7 | sampleRate == 0 / hopSize == 0 | ArgumentOutOfRangeException | Calculate_ZeroSampleRate/HopSize_Throws |

### 実装するテストメソッド

全テストメソッドで `TimingCalculator.Calculate()` の `puaTokenMapper` 引数には `null` を渡す（エッジケース検証に PUA 逆引きは不要）。シグネチャ: `Calculate(phonemeIds, durations, phonemeIdMap, puaTokenMapper: null, sampleRate, hopSize)`。

1. **Calculate_EmptyInput_ReturnsEmpty**
   - 入力: phonemeIds = [], durations = []
   - 検証: entries は空リスト、例外なし
   - piper-plus CalculateTiming_EmptyInput と対応

2. **Calculate_NullPhonemeIds_ThrowsArgumentNullException**
   - 入力: phonemeIds = null
   - 検証: ArgumentNullException がスローされる

3. **Calculate_NullDurations_ThrowsArgumentNullException**
   - 入力: durations = null
   - 検証: ArgumentNullException がスローされる

4. **Calculate_NullPhonemeIdMap_ThrowsArgumentNullException**
   - 入力: phonemeIdMap = null
   - 検証: ArgumentNullException がスローされる

5. **Calculate_LengthMismatch_PhonemeIdsLonger_UsesMinLength**
   - 入力: phonemeIds = [10, 12, 10], durations = [5.0f, 3.0f]（phonemeIds が長い）
   - 検証: entries.Length == 2（Math.Min で打ち切り）
   - E2 カバレッジ

6. **Calculate_LengthMismatch_DurationsLonger_UsesMinLength**
   - 入力: phonemeIds = [10], durations = [5.0f, 3.0f, 2.0f]（durations が長い）
   - 検証: entries.Length == 1（Math.Min で打ち切り）
   - E2 カバレッジ

7. **Calculate_AllZeroDurations_NoCrash**
   - 入力: phonemeIds = [10, 12], durations = [0.0f, 0.0f]
   - 検証: entries.Length == 2、全エントリの DurationSeconds == 0（算出プロパティ: EndSeconds - StartSeconds == 0）、例外なし
   - E3 カバレッジ

8. **Calculate_NegativeDurations_ClampsToZero**
   - 入力: phonemeIds = [10, 12], durations = [-5.0f, 3.0f]
   - 検証: entries[0].DurationSeconds >= 0（負値がクランプされている。DurationSeconds は算出プロパティなので EndSeconds - StartSeconds >= 0 と等価）
   - 検証: entries[1].StartSeconds >= entries[0].EndSeconds（時間の逆転なし）
   - E4 カバレッジ

9. **Calculate_ZeroSampleRate_ThrowsArgumentOutOfRangeException**
   - 入力: sampleRate = 0
   - 検証: ArgumentOutOfRangeException がスローされる
   - E7 カバレッジ

10. **Calculate_ZeroHopSize_ThrowsArgumentOutOfRangeException**
    - 入力: hopSize = 0
    - 検証: ArgumentOutOfRangeException がスローされる
    - E7 カバレッジ

11. **Calculate_NegativeSampleRate_ThrowsArgumentOutOfRangeException**
    - 入力: sampleRate = -1
    - 検証: ArgumentOutOfRangeException がスローされる

12. **Calculate_NegativeHopSize_ThrowsArgumentOutOfRangeException**
    - 入力: hopSize = -1
    - 検証: ArgumentOutOfRangeException がスローされる

13. **Calculate_VeryLargeDurations_NoOverflow**
    - 入力: durations = [float.MaxValue / 2, 1.0f]
    - 検証: 例外なし、endSeconds が NaN/Infinity でないこと

## 5. 懸念事項とレビュー項目

### 5.1 懸念事項

1. **負 durations のクランプ方式**: piper-plus の TimingWriter は負値クランプを明示的には行っていない（E4 は uPiper 独自要件）。TimingCalculator の実装で `Math.Max(0, frameDuration)` が入るか、テストが先行して実装の制約となる可能性
2. **長さ不一致時の警告ログ**: E2 では `Math.Min` 打ち切りに加えて警告ログ出力が期待される。テストで `Debug.LogWarning` の呼び出しを検証するか、結果の正確性のみを検証するかの判断が必要
3. **float 精度**: AllZeroDurations のテストで DurationSeconds == 0 の厳密比較は安全だが、NegativeDurations のクランプ後の値の比較では delta が必要になる可能性

### 5.2 レビューチェックリスト

- [ ] 要件定義 E2-E8 のうち TimingCalculator 責務のケース（E2, E3, E4, E6, E7）が全てカバーされているか
- [ ] Assert.Throws / Assert.ThrowsAsync の使い分けが正しいか（Calculate は同期メソッドのため Assert.Throws を使用）
- [ ] null テストが phonemeIds, durations, phonemeIdMap の3引数全てに対して存在するか（puaTokenMapper は nullable 設計のため null テスト不要）
- [ ] 長さ不一致テストが両方向（phonemeIds > durations, durations > phonemeIds）で存在するか
- [ ] piper-plus の CalculateTiming_EmptyInput と同等のテストがあるか
- [ ] ShortTextProcessorTests の NeedsPadding_NullInput_ThrowsArgumentNullException と同じパターンで null テストが書かれているか

## 6. 一から作り直すとしたら

エッジケーステストは防御的プログラミングの検証であるため、テスト先行（TDD）で書くのが最適。P1-4 の実装前に本チケットのテストケースリストを確定させ、TimingCalculator の API 契約（何が例外で何がクランプか）を明確にする。特に E4（負 durations のクランプ）は piper-plus にない uPiper 独自要件のため、実装チケット P1-4 と同時に仕様を確定させる必要がある。
