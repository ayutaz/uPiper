# P2-1: IInferenceAudioGenerator 戻り値型変更

## 1. タスク目的とゴール

`IInferenceAudioGenerator.GenerateAudioAsync` の戻り値型を `Task<NativeArray<float>>` から `Task<InferenceOutput>` に変更し、durations テンソルを下流に伝搬できるようにする。同時に、`InferenceOutput` が `internal` 型であるため、`IInferenceAudioGenerator` のアクセス修飾子を `public` から `internal` に変更する。

### なぜ必要か

- Phase 1（P1-2）で定義した `InferenceOutput`（`internal readonly struct`）を推論パイプラインの戻り値として使用するには、インターフェースの戻り値型を変更する必要がある
- `InferenceOutput` は `internal` のため、`public interface` の戻り値として使用するとコンパイルエラー（CS0050: inconsistent accessibility）が発生する
- `IInferenceAudioGenerator` の実装クラス（`InferenceAudioGenerator`、`ShortTextMitigatingGenerator`）は全て `internal` / `public class`（ただし内部使用のみ）であり、公開APIは `IPiperTTS` 経由で提供される。インターフェース自体を `internal` にしても外部利用者への影響はない

## 2. 実装する内容の詳細

### 2.0 前提作業: InferenceOutput の sealed class 化

**[Phase 1 レビュー指摘からの持ち越し]**

P2-1 の実装に先立ち、`InferenceOutput` を `internal readonly struct` から `internal sealed class` に変更する。

**理由**: `readonly struct` + `IDisposable` は defensive copy リスクがある。`readonly` フィールドに格納された `InferenceOutput` に対して `Dispose()` を呼ぶと defensive copy が作られ、元の NativeArray が破棄されない。Phase 2 以降で実際に使用される（`ExecuteInference` の戻り値、`using var output = ...` パターン）ため、Phase 2 開始時点で解消する。

**変更内容**（`Assets/uPiper/Runtime/Core/AudioGeneration/InferenceOutput.cs`）:
```csharp
// Before:
internal readonly struct InferenceOutput : IDisposable
{
    public NativeArray<float> Audio { get; }
    public NativeArray<float> Durations { get; }
    public bool HasDurations => Durations.IsCreated;
    // ...
    public void Dispose() { /* IsCreated チェック後に Dispose */ }
}

// After:
internal sealed class InferenceOutput : IDisposable
{
    private bool _disposed;
    public NativeArray<float> Audio { get; }
    public NativeArray<float> Durations { get; }
    public bool HasDurations => Durations.IsCreated;

    internal InferenceOutput(NativeArray<float> audio, NativeArray<float> durations)
    {
        Audio = audio;
        Durations = durations;
    }

    public void Dispose()
    {
        if (_disposed) return;
        _disposed = true;
        if (Audio.IsCreated) Audio.Dispose();
        if (Durations.IsCreated) Durations.Dispose();
    }
}
```

**メリット**:
- defensive copy 問題の完全解消
- `_disposed` フラグによる二重 Dispose 防止
- `InferenceContext`（既存の sealed class + IDisposable パターン）との整合

**デメリット**:
- ヒープアロケーション1回（GC 圧力）。ただし Gen0 即回収のため実害は軽微（生存期間は推論メソッド内の数行）

**影響範囲**: P2-1 の実装変更対象ファイルに `InferenceOutput.cs` を追加。`default(InferenceOutput)` を使用していた箇所は `null` チェックに変更する。T2-2 のテストも sealed class 前提で実装する。

### 2.2 IInferenceAudioGenerator.cs の変更

```csharp
// Before:
public interface IInferenceAudioGenerator : IDisposable
{
    public Task<NativeArray<float>> GenerateAudioAsync(
        int[] phonemeIds,
        int[] prosodyFlat = null,
        float lengthScale = 1.0f,
        float noiseScale = 0.667f,
        float noiseW = 0.8f,
        int speakerId = 0,
        int languageId = 0,
        CancellationToken cancellationToken = default);
    // ...
}

// After:
internal interface IInferenceAudioGenerator : IDisposable
{
    /// <summary>
    /// 音素から音声を生成する（Prosody対応統合版）。
    /// prosodyFlatがnullの場合はProsodyなしで推論する。
    /// </summary>
    /// <remarks>Caller owns and must Dispose the returned InferenceOutput.</remarks>
    public Task<InferenceOutput> GenerateAudioAsync(
        int[] phonemeIds,
        int[] prosodyFlat = null,
        float lengthScale = 1.0f,
        float noiseScale = 0.667f,
        float noiseW = 0.8f,
        int speakerId = 0,
        int languageId = 0,
        CancellationToken cancellationToken = default);
    // ...（他のメンバーは変更なし）
}
```

**変更点**:
1. `public interface` → `internal interface`
2. `Task<NativeArray<float>>` → `Task<InferenceOutput>`
3. XMLドキュメントコメントの `<remarks>` を `NativeArray` から `InferenceOutput` に更新
4. `internal interface` では `public` メンバー修飾子は不要だが、既存スタイルとの一貫性から維持してもよい（コンパイルには影響しない）

// Before:
private NativeArray<float> ExecuteInference(...)
{
    NativeArray<float> audioData = default;
    try
    {
        audioData = ExtractResults();

            audioData.Dispose();

**注意**: `ExtractResults()` の戻り値型変更は P2-2 のスコープ。P2-1 では暫定的に `new InferenceOutput(audioData, default)` でラップする。

### 2.4 暫定ラップ（P2-2 前の中間状態）

P2-1 完了時点では `ExtractResults()` はまだ `NativeArray<float>` を返す。P2-2 で `InferenceOutput` を返すように変更されるまでの中間状態として:

```csharp
// P2-1 暫定: ExtractResults() は NativeArray<float> を返す
// 注: §2.0 により InferenceOutput は sealed class に変更済み。
//      default ではなく null チェックを使用する。
private InferenceOutput ExecuteInference(...)
{
    InferenceOutput output = null;
    try
    {
        var audioData = ExtractResults(); // まだ NativeArray<float>
        output = new InferenceOutput(audioData, default); // durations なし
        return output;
    }
    catch (Exception ex)
    {
        output?.Dispose();
        PiperLogger.LogError($"[InferenceAudioGenerator] Failed to execute inference: {ex.Message}");
        throw;
    }
}
```

### 2.5 SplitInferenceOrchestrator.cs の変更

```csharp
// L80-84:
// Before:
var phraseAudio = await _generator.GenerateAudioAsync(
    phrase.PhonemeIds, phrase.ProsodyFlat,
    lengthScale, noiseScale, noiseW,
    speakerId, languageId, cancellationToken);
segments.Add((phraseAudio, phrase.SilenceSamples));
totalLength += phraseAudio.Length + phrase.SilenceSamples;

// After:
using var phraseOutput = await _generator.GenerateAudioAsync(
    phrase.PhonemeIds, phrase.ProsodyFlat,
    lengthScale, noiseScale, noiseW,
    speakerId, languageId, cancellationToken);
// Audio を取り出し、InferenceOutput.Durations は句分割では使用しない（P3-1 で対応）
var phraseAudio = new NativeArray<float>(phraseOutput.Audio.Length, Allocator.Persistent);
NativeArray<float>.Copy(phraseOutput.Audio, phraseAudio);
segments.Add((phraseAudio, phrase.SilenceSamples));
totalLength += phraseAudio.Length + phrase.SilenceSamples;
```

**注意**: P3-1 で SplitInferenceOrchestrator が durations を結合するロジックを追加するため、ここでは Audio のみ抽出する暫定実装とする。Audioのコピーが必要な理由は、`using var` で `phraseOutput` を Dispose すると内部の Audio NativeArray も破棄されるため。

**代替案（コピー回避）**: InferenceOutput の Audio を直接取り出し、Durations のみ Dispose するヘルパーを設ける方法もあるが、P3-1 で durations 結合ロジックが追加される時点でリファクタリングされるため、P2-1 ではシンプルなコピーで対応する。

## 6. 一から作り直すとしたら

### 代替案A: InferenceOutput を public にする

```csharp
public readonly struct InferenceOutput : IDisposable { ... }
public interface IInferenceAudioGenerator : IDisposable { ... }
```

**メリット**: アクセス修飾子の変更が不要。外部パッケージから推論レイヤーを直接利用可能。
**デメリット**: `NativeArray<float>` の Dispose 責務が外部に露出する。内部実装の詳細（durations テンソル形式）が公開APIの一部になる。将来の型変更が破壊的変更になる。

**判定**: 推論レイヤーは内部実装詳細であり、公開APIは `IPiperTTS` + `SynthesisWithTimingResult`（P1-3）で提供する設計方針に従い、`internal` を維持する。

### 代替案B: IInferenceAudioGenerator を分割

```csharp
// 既存インターフェース（public、互換性維持）
public interface IInferenceAudioGenerator : IDisposable
{
    Task<NativeArray<float>> GenerateAudioAsync(...);
}

// 新規インターフェース（internal、durations 対応）
internal interface IInferenceAudioGeneratorWithDurations : IInferenceAudioGenerator
{
    Task<InferenceOutput> GenerateAudioWithDurationsAsync(...);
}
```

**メリット**: 既存の `public` インターフェースを維持。後方互換性を完全に保証。
**デメリット**: インターフェース階層が複雑化。呼び出し側でキャストが必要。`IInferenceAudioGenerator` の全実装クラスが `internal` であるため、公開インターフェースを維持する実益がない。

**判定**: 複雑性に見合うメリットがなく不採用。`IInferenceAudioGenerator` の全利用者が同一アセンブリ内であることから、`internal` への変更が最もシンプル。

### 代替案C: GenerateAudioAsync に out パラメータを追加

```csharp
Task<NativeArray<float>> GenerateAudioAsync(
    ...,
    out NativeArray<float> durations);
```

**メリット**: 戻り値型の変更が不要。
**デメリット**: `async` メソッドで `out` パラメータは使用不可（C# 言語仕様制約）。

**判定**: 言語制約により不可能。

## 7. 後続タスクへの連絡事項

### P2-2 (InferenceAudioGenerator durations 読み取り) への連絡

- `ExecuteInference` は `InferenceOutput` を返すようになっている。暫定で `new InferenceOutput(audioData, default)` でラップしているため、`ExtractResults()` を `InferenceOutput` を返すように変更し、durations の読み取りを追加すること
- `InferenceAudioGenerator` は `internal class` に変更済み
- 例外パスの Dispose は `output.Dispose()` に集約済み。durations 追加後も同じパターンで安全

### P2-3 (ShortTextMitigatingGenerator 追従) への連絡

- `GenerateAudioAsync` の戻り値型は `Task<InferenceOutput>` に変更済み（P2-1 で最小限変更として実施）
- P2-1 時点の暫定実装: `_inner.GenerateAudioAsync()` の戻り値 `InferenceOutput` から `.Audio` を取り出し、既存のトリムロジックを適用後、`new InferenceOutput(trimmedAudio, default)` で再ラップ
- P2-3 では durations の PAD 除去ロジックを追加し、トリム後の `InferenceOutput` に正しい durations を含めること

### P2-4 (StubInferenceAudioGenerator durations 対応) への連絡

- `GenerateAudioAsync` の戻り値型は `Task<InferenceOutput>` に変更済み（P2-1 で最小限変更として実施）
- P2-1 時点の暫定実装: `new InferenceOutput(nativeArray, default)`
- P2-4 では `DurationsToReturn` プロパティを追加し、`InferenceOutput` に durations を含めて返却すること

### P3-1 (SplitInferenceOrchestrator durations 結合) への連絡

- P2-1 では SplitInferenceOrchestrator 内で `InferenceOutput` から Audio のみ抽出し、Durations は破棄する暫定実装になっている
- P3-1 で durations 結合ロジックを追加する際に、この暫定コードをリファクタリングすること
- Audio のコピーオーバーヘッドも P3-1 でのリファクタリングで解消可能（Audio の所有権移転パターン等）
