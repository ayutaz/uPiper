# T2-1: ShortTextMitigatingGenerator PAD 除去テスト

## 1. タスク目的とゴール

ShortTextMitigatingGenerator が短テキスト緩和（Strategy A）で PAD パディングした際に、InferenceOutput.Durations から PAD 対応要素を正しく除去するロジックを検証する。P2-3 で ShortTextMitigatingGenerator の戻り値が `InferenceOutput` に変更され、durations の PAD 除去ロジックが追加される。本チケットではそのロジックの正確性を、StubInferenceAudioGenerator の `DurationsToReturn` 経由で固定 durations を注入して検証する。

**ゴール**: PAD パディング → 推論 → durations PAD 除去の一連のフローで、除去後の durations 要素数が元の実音素数と一致し、実音素に対応する duration 値が保持されていることを保証する。

### PAD パディングロジック（前提知識）

ShortTextProcessor.PadPhonemeIds のパディング配置:
```
deficit = MinPhonemeIds(40) - phonemeIds.Length
afterBos = deficit / 2
beforeEos = deficit - afterBos

構造: [BOS, PAD*afterBos, body..., PAD*beforeEos, EOS]
```

P2-3 で追加される durations PAD 除去ロジック:
- パディング前の `phonemeIds.Length` を記録
- 推論後、`InferenceOutput.Durations` からパディング位置の要素を除去
- 除去対象: BOS 直後の `afterBos` 個 + EOS 直前の `beforeEos` 個
- 除去後の durations.Length == 元の phonemeIds.Length

### 実装するテストメソッド

1. **GenerateAudioAsync_ShortInput_DurationsStrippedOfPadding**
   - 入力: phonemeIds = [BOS=1, 10, 11, 12, EOS=2]（5要素、deficit=35）
   - StubGenerator: AudioDataToReturn = float[100]（ダミー）, DurationsToReturn = float[40]（全要素にインデックス値を設定）
   - 呼び出し: `_mitigator.GenerateAudioAsync(phonemeIds)`
   - 検証: `output.Durations.Length == 5`（元の phonemeIds.Length）
   - 検証: 除去後の durations がパディング前の実音素位置に対応する値のみを含む
   - NativeArray は try/finally で Dispose

2. **GenerateAudioAsync_ShortInput_DurationsCountMatchesOriginal**
   - 入力: phonemeIds = [BOS=1, 10, 11, 12, 13, 14, 15, 16, 17, EOS=2]（10要素、deficit=30）
   - StubGenerator: DurationsToReturn = float[40]（連番値）
   - 検証: `output.Durations.Length == 10`
   - 元の実音素に対応する duration 値が正確に保持されていることをインデックスベースで検証

3. **GenerateAudioAsync_ShortInput_BosEosDurationsPreserved**
   - 入力: phonemeIds = [BOS=1, 10, EOS=2]（3要素、deficit=37）
   - StubGenerator: DurationsToReturn = float[40]
   - DurationsToReturn[0] = 5.0f（BOS）, DurationsToReturn[39] = 3.0f（EOS）
   - Body 位置（afterBos=18 の次、インデックス=19）に 10.0f を設定
   - 検証: output.Durations[0] == 5.0f（BOS）
   - 検証: output.Durations[1] == 10.0f（body の音素 10）
   - 検証: output.Durations[2] == 3.0f（EOS）

4. **GenerateAudioAsync_ExactMinimum_NoPaddingNoDurationsChange**
   - 入力: phonemeIds = int[40]（ちょうど MinPhonemeIds、パディング不要）
   - StubGenerator: DurationsToReturn = float[40]（連番値）
   - 検証: `output.Durations.Length == 40`（変更なし）
   - 検証: durations の全要素が元の値と一致

5. **GenerateAudioAsync_LongInput_NoPaddingNoDurationsChange**
   - 入力: phonemeIds = int[60]（MinPhonemeIds 超え）
   - StubGenerator: DurationsToReturn = float[60]（連番値）
   - 検証: `output.Durations.Length == 60`（変更なし）

6. **GenerateAudioAsync_ShortInput_NoDurations_HasDurationsFalse**
   - 入力: phonemeIds = [BOS=1, 10, 11, EOS=2]（4要素、パディング対象）
   - StubGenerator: DurationsToReturn = null（durations 非対応モデル）
   - 検証: `output.HasDurations == false`
   - durations が存在しない場合、PAD 除去ロジックがスキップされ、例外が発生しないこと

7. **GenerateAudioAsync_ShortInput_EvenDeficit_SymmetricPadRemoval**
   - 入力: phonemeIds = int[20]（deficit=20、afterBos=10, beforeEos=10）
   - StubGenerator: DurationsToReturn = float[40]
   - 偶数 deficit のケースで afterBos == beforeEos であることを前提に、対称的な除去を検証
   - 検証: `output.Durations.Length == 20`

8. **GenerateAudioAsync_ShortInput_OddDeficit_AsymmetricPadRemoval**
   - 入力: phonemeIds = int[21]（deficit=19、afterBos=9, beforeEos=10）
   - StubGenerator: DurationsToReturn = float[40]
   - 奇数 deficit のケースで afterBos != beforeEos であることを検証
   - 検証: `output.Durations.Length == 21`
   - 検証: 除去されたのが正しい位置（BOS直後の9個 + EOS直前の10個）であること

### ヘルパーメソッド

```csharp
/// <summary>
/// BOS + body + EOS の構造を持つ phonemeIds を構築する。
/// Body 部は連番（10, 11, 12, ...）。
/// </summary>
private static int[] CreatePhonemeIds(int length)
{
    if (length < 2) return new[] { BosId };
    var ids = new int[length];
    ids[0] = BosId;
    ids[length - 1] = EosId;
    for (var i = 1; i < length - 1; i++)
        ids[i] = 10 + i - 1;
    return ids;
}

/// <summary>
/// 連番の durations 配列を構築する。
/// durations[i] = (float)(i + 1) で一意性を保証。
/// </summary>
private static float[] CreateSequentialDurations(int length)
{
    var durations = new float[length];
    for (var i = 0; i < length; i++)
        durations[i] = (float)(i + 1);
    return durations;
}
```

## 6. 一から作り直すとしたら

テスト設計は「PAD 除去の入出力検証」に絞るのが最適。ShortTextMitigatingGenerator 内部の実装詳細（TrimSilence, AdjustScales）は既存の ShortTextProcessorTests.cs で十分にカバーされているため、本チケットでは durations の PAD 除去に特化する。StubInferenceAudioGenerator 経由で durations を注入するアプローチは、ONNX 推論への依存を排除しつつ PAD 除去ロジックを isolated にテストできるため、変更不要。

deficit の偶数/奇数で afterBos/beforeEos の非対称性が生じる点は、ShortTextProcessor.PadPhonemeIds と同じロジック（`deficit / 2` の整数除算）を使用しているため、テストでもこの計算式を明示的に検証する設計が適切。

## 7. 後続タスクへの連絡事項

- P2-3 実装時: ShortTextMitigatingGenerator の `GenerateAudioAsync` で、パディング前の `phonemeIds.Length` を変数に保持し、推論後の `InferenceOutput.Durations` から PAD 位置の要素を除去して新しい `InferenceOutput` を構築すること。除去位置は `afterBos = deficit / 2`（BOS 直後）と `beforeEos = deficit - afterBos`（EOS 直前）

- 既存テストファイル（ShortTextProcessorTests.cs, TTSSynthesisOrchestratorTests.cs）への変更は不要。P2-1/P2-3 で戻り値型が変更される際に、既存テストの `NativeArray<float>` を `InferenceOutput` に適応させるのは各実装チケットの責務
