# P1-4: TimingCalculator 実装

## 1. タスク目的とゴール

ONNX モデルが出力する `durations` テンソル（各音素のフレーム持続時間）を秒単位のタイミング情報に変換する `TimingCalculator` を実装する。piper-plus の `TimingWriter.CalculateTiming` (L70-120) を Unity/uPiper 向けに移植し、PAD/BOS/EOS スキップ、PUA 逆引き、エッジケース防御を含む堅牢な変換ロジックを提供する。

**ゴール**: `TimingCalculator.Calculate()` が正しいタイミング情報を返却し、関連テストチケット (T1-1 ~ T1-4) の全テストをパスすること。

## 2. 実装する内容の詳細

### 2.1 新規ファイル

**パス**: `Assets/uPiper/Runtime/Core/AudioGeneration/TimingCalculator.cs`

### 2.2 クラス設計

```csharp
using System;
using System.Collections.Generic;
using uPiper.Core.Logging;
using uPiper.Core.Phonemizers.Multilingual;

namespace uPiper.Core.AudioGeneration
{
    /// <summary>
    /// ONNX モデルの durations テンソルから各音素の開始・終了時刻を算出する。
    /// piper-plus TimingWriter.CalculateTiming の Unity 向け移植。
    /// </summary>
    public static class TimingCalculator
    {
        // 特殊トークン ID (PhonemeEncoder と一致)
        private const int PadId = 0;
        private const int BosId = 1;
        private const int EosId = 2;

        /// <summary>
        /// 音素 ID 配列と durations 配列からタイミング情報を算出する。
        /// </summary>
        /// <param name="phonemeIds">
        /// モデルに入力した音素 ID 配列。PhonemeEncoder.Encode() の出力と同一順序。
        /// </param>
        /// <param name="durations">
        /// ONNX モデルの durations 出力テンソル。
        /// durations[i] は phonemeIds[i] のスペクトログラムフレーム数。
        /// </param>
        /// <param name="phonemeIdMap">
        /// モデル設定 (onnx.json) 由来の phoneme_id_map。
        /// キー: 音素文字列、値: int[] (先頭要素が ID)。
        /// </param>
        /// <param name="puaTokenMapper">
        /// PUA 逆引き用マッパー。PUA 文字を多文字トークンに変換する。
        /// null の場合は PUA 逆引きをスキップする。
        /// </param>
        /// <param name="sampleRate">
        /// オーディオサンプルレート (Hz)。通常 22050。
        /// </param>
        /// <param name="hopSize">
        /// スペクトログラムのホップサイズ (サンプル数)。VITS デフォルト: 256。
        /// </param>
        /// <returns>
        /// PAD/BOS/EOS を除く実音素のタイミングリスト。
        /// 空入力の場合は空リストを返す。
        /// </returns>
        /// <exception cref="ArgumentNullException">
        /// phonemeIds, durations, phonemeIdMap のいずれかが null の場合。
        /// </exception>
        /// <exception cref="ArgumentOutOfRangeException">
        /// sampleRate または hopSize が 0 以下の場合。
        /// </exception>
        public static List<PhonemeTimingEntry> Calculate(
            int[] phonemeIds,
            float[] durations,
            Dictionary<string, int[]> phonemeIdMap,
            PuaTokenMapper puaTokenMapper,
            int sampleRate,
            int hopSize = 256)
        {
            // --- バリデーション ---
            if (phonemeIds == null)
                throw new ArgumentNullException(nameof(phonemeIds));
            if (durations == null)
                throw new ArgumentNullException(nameof(durations));
            if (phonemeIdMap == null)
                throw new ArgumentNullException(nameof(phonemeIdMap));
            if (sampleRate <= 0)
                throw new ArgumentOutOfRangeException(
                    nameof(sampleRate),
                    sampleRate,
                    "Sample rate must be positive.");
            if (hopSize <= 0)
                throw new ArgumentOutOfRangeException(
                    nameof(hopSize),
                    hopSize,
                    "Hop size must be positive.");

            // 空入力 → 空リスト
            if (phonemeIds.Length == 0 || durations.Length == 0)
                return new List<PhonemeTimingEntry>();

            // --- 逆引き辞書構築 ---
            var reverseIdMap = BuildReverseIdMap(phonemeIdMap, puaTokenMapper);

            // --- タイミング計算 ---
            float frameLength = (float)hopSize / sampleRate;
            int count = Math.Min(phonemeIds.Length, durations.Length);
            var entries = new List<PhonemeTimingEntry>(count);
            float currentTime = 0f;

            if (phonemeIds.Length != durations.Length)
            {
                PiperLogger.LogWarning(
                    $"[TimingCalculator] phonemeIds.Length ({phonemeIds.Length}) != " +
                    $"durations.Length ({durations.Length}). " +
                    $"Processing min({phonemeIds.Length}, {durations.Length}) = {count} elements.");
            }

            for (int i = 0; i < count; i++)
            {
                int id = phonemeIds[i];
                float frameDuration = durations[i];

                // 負の duration はクランプ
                if (frameDuration < 0f)
                {
                    PiperLogger.LogWarning(
                        $"[TimingCalculator] Negative duration ({frameDuration}) " +
                        $"at index {i} (phonemeId={id}). Clamping to 0.");
                    frameDuration = 0f;
                }

                // PAD(0) / BOS(1) / EOS(2) はスキップ（時間カーソルは進行）
                if (id is PadId or BosId or EosId)
                {
                    currentTime += frameDuration * frameLength;
                    continue;
                }

                // 通常音素: タイミングエントリ生成
                float startTime = currentTime;
                currentTime += frameDuration * frameLength;
                float endTime = currentTime;

                string phonemeStr = ResolvePhonemeString(id, reverseIdMap);

                entries.Add(new PhonemeTimingEntry(
                    phonemeStr,
                    startTime,
                    endTime));
            }

            return entries;
        }

        /// <summary>
        /// phonemeIdMap から逆引き辞書 (ID → 表示用音素文字列) を構築する。
        /// PUA 文字キーは puaTokenMapper で多文字トークンに変換する。
        /// </summary>
        internal static Dictionary<int, string> BuildReverseIdMap(
            Dictionary<string, int[]> phonemeIdMap,
            PuaTokenMapper puaTokenMapper)
        {
            var reverse = new Dictionary<int, string>(phonemeIdMap.Count);
            foreach (var (phonemeStr, ids) in phonemeIdMap)
            {
                if (ids is not { Length: > 0 })
                    continue;

                string display = phonemeStr;

                // PUA 単文字キーを多文字トークンに逆引き
                if (phonemeStr.Length == 1 && puaTokenMapper != null)
                {
                    var token = puaTokenMapper.UnmapChar(phonemeStr[0]);
                    if (token != null)
                    {
                        display = token;
                    }
                }

                reverse.TryAdd(ids[0], display);
            }

            return reverse;
        }

        /// <summary>
        /// 音素 ID を表示用文字列に解決する。
        /// 未知の ID は "?" を返す (piper-plus の UNKNOWN_PHONEME と同等)。
        /// </summary>
        private static string ResolvePhonemeString(
            int id,
            Dictionary<int, string> idToString)
        {
            if (idToString.TryGetValue(id, out var str))
                return str;

            // フォールバック: 印字可能 ASCII 範囲なら文字そのもの
            if (id is > 2 and < 128)
                return ((char)id).ToString();

            return "?";
        }
    }
}
```

### 2.3 アルゴリズム詳細

**piper-plus CalculateTiming (TimingWriter.cs L70-120) からの移植ポイント**:

| ステップ | 処理 | piper-plus 対応行 |
|---------|------|-------------------|
| 1 | `frameLength = (float)hopSize / sampleRate` | L89: `float frameLength = (float)hopSize / sampleRate;` |
| 2 | `count = Math.Min(phonemeIds.Length, durations.Length)` | L91: `int count = Math.Min(phonemeIds.Length, durations.Length);` |
| 3 | ループ: PAD(0)/BOS(1)/EOS(2) スキップ | L99-104: `if (id is 0 or 1 or 2) { currentTime += ...; continue; }` |
| 4 | 通常 ID: `startTime = currentTime; currentTime += frameDuration * frameLength; endTime = currentTime;` | L106-108 |
| 5 | 逆引き辞書で音素文字列取得 | L110: `ResolvePhonemeString(id, idToString)` |
| 6 | PhonemeTimingEntry 生成 | L112-116 |

**uPiper 固有の差分**:

| 差分 | piper-plus | uPiper | 理由 |
|------|-----------|--------|------|
| phonemeIds の型 | `long[]` | `int[]` | uPiper の PhonemeEncoder.Encode() が int[] を返すため |
| PUA 逆引き | `Mapping.OpenJTalkToPiperMapping.CharToToken` (静的辞書) | `PuaTokenMapper.UnmapChar()` (インスタンスメソッド) | uPiper は PuaTokenMapper をランタイムに初期化し、pua.json で動的拡張可能 |
| 負 duration 防御 | なし | クランプ + 警告ログ | ONNX 推論の数値不安定性への防御 |
| 長さ不一致防御 | Math.Min のみ | Math.Min + 警告ログ | デバッグ容易性向上 |

### 2.4 GC アロケーション

- `List<PhonemeTimingEntry>`: 1回のみ確保 (`capacity = count`)
- `Dictionary<int, string>` (逆引き辞書): 1回のみ確保。Calculate() 呼び出しごとに構築
  - 将来の最適化候補: 逆引き辞書のキャッシュ（PhonemeIdMap が同一の場合は再利用可能）。ただし Phase 1 ではシンプルさを優先
- `ResolvePhonemeString` の `((char)id).ToString()`: フォールバックパスのみ。通常パスでは辞書ヒットするためアロケーションなし
- PhonemeTimingEntry: readonly struct のため boxing なし (`List<T>` は generic なので boxing 不要)

### 2.5 エッジケース一覧

| ケース | 入力例 | 期待動作 |
|--------|--------|---------|
| 空 phonemeIds | `new int[0]` | 空リスト返却 |
| 空 durations | `new float[0]` | 空リスト返却 |
| null phonemeIds | `null` | ArgumentNullException |
| null durations | `null` | ArgumentNullException |
| null phonemeIdMap | `null` | ArgumentNullException |
| sampleRate <= 0 | `0`, `-1` | ArgumentOutOfRangeException |

## 5. 懸念事項とレビュー項目

### 5.1 懸念事項

| # | 懸念 | 対策 |
|---|------|------|
| 1 | phonemeIds が int[] だが piper-plus は long[] | uPiper の PhonemeEncoder が int[] を返すため int[] で統一。音素 ID は通常 0-100 程度なので int で十分。PhonemeIdMap の値も int[] |
| 2 | BuildReverseIdMap を毎回構築するコスト | Phase 1 ではシンプルさを優先。phonemeIdMap は通常 50-100 エントリなのでマイクロ秒オーダー。頻繁呼び出しが問題になれば Phase 3 以降でキャッシュ化 |
| 3 | PuaTokenMapper がまだ初期化されていない場合 | puaTokenMapper パラメータを nullable にし、null の場合は PUA 逆引きをスキップする。phonemeIdMap のキー文字列がそのまま表示される |
| 4 | NaN / Infinity の duration | float の特殊値は `< 0f` チェックでは捕捉されない。ただし NaN/Infinity は ONNX 推論の致命的エラーを示すため、TimingCalculator レベルでの防御は過剰。推論レイヤー (P2-2) で検証すべき |
| 5 | ShortTextMitigatingGenerator が PAD を挿入した場合 | PAD (ID=0) は自動スキップされるため問題なし。durations 配列も同じ長さに拡張されている前提 (P2-3 で保証) |

## 6. 一から作り直すとしたら

現設計は piper-plus の実績あるアルゴリズムの忠実な移植であり、基本構造は妥当。再設計する場合に検討する点:

- **逆引き辞書のキャッシュ**: `Calculate()` を static メソッドではなくインスタンスメソッドにし、コンストラクタで逆引き辞書を構築・保持する案。ただし AudioNormalizer 等の既存 static ユーティリティとの一貫性が崩れるため、Phase 1 では見送り。必要になった時点でリファクタリングする方が変更コストが低い
- **Span<T> 対応**: `ReadOnlySpan<int>` / `ReadOnlySpan<float>` を引数にすれば NativeArray からの変換時にコピー不要になるが、C# 10.0 + Unity の Span サポート状況を考慮して Phase 1 では配列で十分
- **戻り値の型**: `List<PhonemeTimingEntry>` ではなく `IReadOnlyList<PhonemeTimingEntry>` を返す案もあるが、呼び出し元 (TTSSynthesisOrchestrator) が即座に SynthesisWithTimingResult に格納するため、中間の不変ラッパーは不要。SynthesisWithTimingResult 側で防御的コピーを行う

## 7. 後続タスクへの連絡事項

### T1-1 ~ T1-4 (テストチケット) へ
- `Calculate()` のシグネチャ: `(int[] phonemeIds, float[] durations, Dictionary<string, int[]> phonemeIdMap, PuaTokenMapper puaTokenMapper, int sampleRate, int hopSize = 256)`
- `BuildReverseIdMap()` は `internal static` なので `InternalsVisibleTo` 経由で直接テスト可能
- テストで PuaTokenMapper のモックが必要な場合は `new PuaTokenMapper()` でインスタンス生成可能 (固定マッピングのみ使用)
- frameLength の期待値: `256 / 22050 ≈ 0.011609977f`

### P3-2 (TTSSynthesisOrchestrator タイミング計算統合) へ
- `TimingCalculator.Calculate()` は純粋関数。スレッドセーフ。async 不要
- 呼び出しタイミング: `InferenceAudioGenerator.GenerateAudioAsync()` 完了後、AudioClip 構築前
- 必要な引数:
  - `phonemeIds`: PhonemeEncoder.Encode() / EncodeWithProsody() の出力
  - `durations`: InferenceOutput.Durations (P2-2 で追加)
  - `phonemeIdMap`: PiperVoiceConfig.PhonemeIdMap
  - `puaTokenMapper`: PiperTTS が保持する PuaTokenMapper インスタンス
  - `sampleRate`: IInferenceAudioGenerator.SampleRate
- 戻り値の `List<PhonemeTimingEntry>` はそのまま `SynthesisWithTimingResult` のコンストラクタに渡せる (P1-3 側で防御的コピーする)

### P2-3 (ShortTextMitigatingGenerator 追従) へ
- ShortTextMitigatingGenerator が PAD を挿入した場合、durations 配列も同じ長さに拡張されている必要がある
- TimingCalculator は PAD (ID=0) を自動スキップするため、PAD 挿入分は無害
- ただし、Strategy A の「無音トリム」後に durations 配列の対応要素を除去する必要があるかは P2-3 で検討すること
