# P2-4: StubInferenceAudioGenerator durations 対応

## 1. タスク目的とゴール

テスト用スタブ `StubInferenceAudioGenerator` に durations 出力機能を追加し、Phase 2-4 のテストチケット（T2-1, T3-1, T3-3, T4-1 等）で durations を含む推論結果をシミュレートできるようにする。

### なぜ必要か

- P2-1 で `GenerateAudioAsync` の戻り値が `Task<InferenceOutput>` に変更されたため、スタブも `InferenceOutput` を返す必要がある
- 後続テスト（T2-1: ShortTextMitigatingGenerator PAD 除去テスト、T3-1: SplitInferenceOrchestrator 句分割タイミングテスト等）で、既知の durations データを設定してタイミング計算の正確性を検証する必要がある
- `StubInferenceAudioGenerator` はテスト専用のため、`DurationsToReturn` プロパティで任意の durations データを注入可能にする

## 2. 実装する内容の詳細

### 2.1 変更対象ファイル

**パス**: `Assets/uPiper/Tests/Editor/AudioGeneration/StubInferenceAudioGenerator.cs`

### 2.2 新規プロパティ

/// GenerateAudioAsync が返す固定 durations データ（managed配列）。
/// null の場合は durations なし（InferenceOutput.HasDurations == false）。

/// テスト設定用。デフォルト: false。
/// </summary>
public bool SupportsDurations { get; set; } = false;

/// <summary>最後の GenerateAudioAsync 呼び出しで返された durations の有無</summary>
public bool LastHadDurations { get; private set; }

### 2.3 GenerateAudioAsync の変更

```csharp
public Task<InferenceOutput> GenerateAudioAsync(
    int[] phonemeIds,
    int[] prosodyFlat = null,
    float lengthScale = 1.0f, float noiseScale = 0.667f, float noiseW = 0.8f,
    int speakerId = 0, int languageId = 0,
    CancellationToken cancellationToken = default)
{
    GenerateCallCount++;
    LastPhonemeIds = phonemeIds;
    LastProsodyFlat = prosodyFlat;
    LastLengthScale = lengthScale;
    LastNoiseScale = noiseScale;
    LastNoiseW = noiseW;
    LastSpeakerId = speakerId;
    LastLanguageId = languageId;

    // Audio
    var audioSource = AudioDataToReturn ?? CreateDefaultAudioData();
    var audioNative = new NativeArray<float>(audioSource.Length, Allocator.Persistent);
    audioNative.CopyFrom(audioSource);

    // Durations
    NativeArray<float> durationsNative = default;
    if (DurationsToReturn != null)
    {
        durationsNative = new NativeArray<float>(DurationsToReturn.Length, Allocator.Persistent);
        durationsNative.CopyFrom(DurationsToReturn);
        LastHadDurations = true;
    }
    else
    {
        LastHadDurations = false;
    }

    var output = new InferenceOutput(audioNative, durationsNative);
    return Task.FromResult(output);
}
```

### 2.4 変更点まとめ

| 変更 | Before | After |
|------|--------|-------|
| `GenerateAudioAsync` 戻り値 | `Task<NativeArray<float>>` | `Task<InferenceOutput>` |
| Audio 返却方式 | `NativeArray<float>` 直接返却 | `InferenceOutput.Audio` に格納 |
| Durations | なし | `DurationsToReturn` が非 null なら `InferenceOutput.Durations` に格納 |
| `SupportsDurations` | P2-2 で追加済み（`{ get; set; } = false`） | 変更なし（P2-2 で追加済み） |
| `LastHadDurations` | なし | 呼び出し記録用プロパティ追加 |

### 2.5 完全な変更後コード

```csharp
using System;
using System.Threading;
using System.Threading.Tasks;
using Unity.Collections;
using Unity.InferenceEngine;
using uPiper.Core;
using uPiper.Core.AudioGeneration;

namespace uPiper.Tests.Editor.AudioGeneration
{
    /// <summary>
    /// IInferenceAudioGenerator のテスト用スタブ実装。
    /// 設定可能な固定audio/durationsデータを返し、呼び出し回数・引数を記録する。
    /// </summary>
    internal sealed class StubInferenceAudioGenerator : IInferenceAudioGenerator
    {
        public bool IsInitialized { get; set; } = true;
        public int SampleRate { get; set; } = 22050;
        public bool SupportsProsody { get; set; } = false;
        public bool SupportsMultiSpeaker { get; set; } = false;
        public bool SupportsLanguageId { get; set; } = false;
        public bool SupportsDurations { get; set; } = false;

        /// <summary>
        /// GenerateAudioAsync が返す固定データ（managed配列）。
        /// null の場合はデフォルト（0.1f x 100 サンプル）。
        /// 呼び出しごとに新しい NativeArray にコピーして返す。
        /// </summary>
        public float[] AudioDataToReturn { get; set; }

        /// <summary>
        /// GenerateAudioAsync が返す固定 durations データ（managed配列）。
        /// null の場合は durations なし（InferenceOutput.HasDurations == false）。
        /// 呼び出しごとに新しい NativeArray にコピーして返す。
        /// </summary>
        public float[] DurationsToReturn { get; set; }

        /// <summary>GenerateAudioAsync の呼び出し回数</summary>
        public int GenerateCallCount { get; private set; }

        /// <summary>最後の GenerateAudioAsync 呼び出し時の phonemeIds</summary>
        public int[] LastPhonemeIds { get; private set; }

        /// <summary>最後の GenerateAudioAsync 呼び出し時の prosodyFlat</summary>
        public int[] LastProsodyFlat { get; private set; }

        /// <summary>最後の GenerateAudioAsync 呼び出し時の lengthScale</summary>
        public float LastLengthScale { get; private set; }

        /// <summary>最後の GenerateAudioAsync 呼び出し時の noiseScale</summary>
        public float LastNoiseScale { get; private set; }

        /// <summary>最後の GenerateAudioAsync 呼び出し時の noiseW</summary>
        public float LastNoiseW { get; private set; }

        /// <summary>最後の GenerateAudioAsync 呼び出し時の speakerId</summary>
        public int LastSpeakerId { get; private set; }

        /// <summary>最後の GenerateAudioAsync 呼び出し時の languageId</summary>
        public int LastLanguageId { get; private set; }

        /// <summary>最後の GenerateAudioAsync 呼び出しで返された durations の有無</summary>
        public bool LastHadDurations { get; private set; }

        public Task InitializeAsync(ModelAsset modelAsset, PiperVoiceConfig config,
            CancellationToken cancellationToken = default)
        {
            IsInitialized = true;
            return Task.CompletedTask;
        }

        public Task InitializeAsync(ModelAsset modelAsset, PiperVoiceConfig config,
            PiperConfig piperConfig, CancellationToken cancellationToken = default)
        {
            IsInitialized = true;
            return Task.CompletedTask;
        }

        public Task<InferenceOutput> GenerateAudioAsync(
            int[] phonemeIds,
            int[] prosodyFlat = null,
            float lengthScale = 1.0f, float noiseScale = 0.667f, float noiseW = 0.8f,
            int speakerId = 0, int languageId = 0,
            CancellationToken cancellationToken = default)
        {
            GenerateCallCount++;
            LastPhonemeIds = phonemeIds;
            LastProsodyFlat = prosodyFlat;
            LastLengthScale = lengthScale;
            LastNoiseScale = noiseScale;
            LastNoiseW = noiseW;
            LastSpeakerId = speakerId;
            LastLanguageId = languageId;

            var audioSource = AudioDataToReturn ?? CreateDefaultAudioData();
            var audioNative = new NativeArray<float>(audioSource.Length, Allocator.Persistent);
            audioNative.CopyFrom(audioSource);

            NativeArray<float> durationsNative = default;
            if (DurationsToReturn != null)
            {
                durationsNative = new NativeArray<float>(
                    DurationsToReturn.Length, Allocator.Persistent);
                durationsNative.CopyFrom(DurationsToReturn);
                LastHadDurations = true;
            }
            else
            {
                LastHadDurations = false;
            }

            var output = new InferenceOutput(audioNative, durationsNative);
            return Task.FromResult(output);
        }

        private static float[] CreateDefaultAudioData()
        {
            var data = new float[100];
            for (var i = 0; i < data.Length; i++)
                data[i] = 0.1f;
            return data;
        }

        public void Dispose() { }
    }
}
```

### 2.6 テストでの使用例

```csharp
// durations 付きのスタブ設定
var stub = new StubInferenceAudioGenerator
{
    AudioDataToReturn = new float[] { 0.1f, 0.2f, 0.3f, /* ... */ },
    DurationsToReturn = new float[] { 5f, 10f, 20f, 3f },
    SupportsDurations = true,
};

var output = await stub.GenerateAudioAsync(
    new[] { 1, 3, 4, 2 }); // BOS, k, o, EOS
using (output)
{
    Assert.IsTrue(output.HasDurations);
    Assert.AreEqual(4, output.Durations.Length);
    Assert.AreEqual(5f, output.Durations[0]);  // BOS duration
    Assert.AreEqual(10f, output.Durations[1]); // k duration
    Assert.AreEqual(20f, output.Durations[2]); // o duration
    Assert.AreEqual(3f, output.Durations[3]);  // EOS duration
}

// durations なしのスタブ設定
var stubNoDurations = new StubInferenceAudioGenerator
{
    // DurationsToReturn = null (デフォルト)
};

var output2 = await stubNoDurations.GenerateAudioAsync(
    new[] { 1, 3, 4, 2 });
using (output2)
{
    Assert.IsFalse(output2.HasDurations);
}
```

### 2.7 既存テストへの影響

`StubInferenceAudioGenerator` を使用している既存テストファイル:
- `SplitInferenceOrchestratorTests.cs`
- `TTSSynthesisOrchestratorTests.cs`

これらのテストは `GenerateAudioAsync` の戻り値を `NativeArray<float>` として受け取っていた。P2-1 で `InferenceOutput` に変更されるため、テスト側も `output.Audio` でアクセスするように修正が必要。

**重要**: P2-1 のスコープで StubInferenceAudioGenerator の戻り値型は最小限変更済み（`new InferenceOutput(audioNative, default)`）。P2-4 では `DurationsToReturn` プロパティの追加と、テストコードの `output.Audio` アクセスへの修正を行う。

既存テストの修正例:
```csharp
// Before:
var audio = await stub.GenerateAudioAsync(phonemeIds);
// audio.Length, audio[i] 等を使用

// After:
using var output = await stub.GenerateAudioAsync(phonemeIds);
var audio = output.Audio;
// audio.Length, audio[i] 等を使用（同じロジック）
// output は using で Dispose される
```

## 6. 一から作り直すとしたら

### 代替案A: Mock フレームワーク（NSubstitute 等）を使用

```csharp
var mock = Substitute.For<IInferenceAudioGenerator>();
mock.GenerateAudioAsync(Arg.Any<int[]>(), ...)
    .Returns(Task.FromResult(new InferenceOutput(audioNative, durationsNative)));
```

**メリット**: スタブクラスの手動メンテナンスが不要。プロパティ追加のたびにスタブを更新する手間がない。
**デメリット**: NSubstitute は Unity パッケージとして追加が必要。テストの可読性が低下（mock 設定が複雑になりがち）。`InferenceOutput` は `internal readonly struct` のため、テストアセンブリでのインスタンス化に `InternalsVisibleTo` が必要な点は同じ。

**判定**: 現状の手動スタブは十分にシンプルであり、プロパティ数も管理可能。Mock フレームワーク導入のコスト対効果が見合わないため不採用。

### 代替案B: StubInferenceAudioGenerator に Builder パターンを追加

```csharp
var stub = new StubInferenceAudioGenerator.Builder()
    .WithAudioData(audioData)
    .WithDurations(durations)
    .WithSupportsDurations(true)
    .Build();
```

**メリット**: テストコードの可読性向上。設定の組み合わせが明示的。
**デメリット**: テスト用スタブに対して過剰な設計。現状のプロパティ直接設定で十分にシンプル。

**判定**: 現状のプロパティ直接設定パターンを維持する。

## 7. 後続タスクへの連絡事項

### T2-1 (ShortTextMitigatingGenerator PAD 除去テスト) への連絡

- `DurationsToReturn` に既知の float[] を設定し、ShortTextMitigatingGenerator 経由で `GenerateAudioAsync` を呼び出して PAD 除去後の durations を検証できる
- テスト例: `DurationsToReturn = new float[] { 1f, 0f, ...(PAD分), 5f, 10f, 0f, ...(PAD分), 2f }`、PAD 除去後に `{ 1f, 5f, 10f, 2f }` が得られることを検証
- `SupportsDurations = true` を設定すること

### T3-1 (SplitInferenceOrchestrator 句分割タイミングテスト) への連絡

- 句ごとに異なる durations を返すには、`GenerateAudioAsync` の呼び出し回数に応じて `DurationsToReturn` を切り替えるか、コールバックを使う必要がある。現状のスタブは固定データを返すため、句分割テストでは各句が同じ durations を受け取る
- 必要に応じて `DurationsToReturn` を `Func<int[], float[]>` 型に拡張するか、呼び出し順に異なるデータを返す `Queue<float[]>` パターンを検討すること

### T3-3 (durations 非対応モデルテスト) への連絡

- `DurationsToReturn = null`（デフォルト）のまま使用すれば durations 非対応モデルをシミュレートできる
- `SupportsDurations = false`（デフォルト）であることも確認すること

### T4-1 (統合テスト: タイミング付き合成) への連絡

- `DurationsToReturn` に `TimingCalculator` で期待通りのタイミングが得られる durations データを設定し、エンドツーエンドの統合テストが可能
- 例: `DurationsToReturn = new float[] { 5f, 10f, 20f, 3f }` で `TimingCalculator.Calculate()` の出力を検証
