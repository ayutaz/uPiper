# T3-2: AudioSynthesisCache タイミング格納テスト

## 1. タスク目的とゴール

AudioSynthesisCache が PhonemeTimingEntry[]（タイミング情報）を音声データと併せてキャッシュに格納・復元できることを検証する。P3-3 で AudioSynthesisCache の `Set` / `TryGet` メソッドが PhonemeTimingEntry[] をオプショナルパラメータとして受け取るように拡張される。本チケットではタイミング情報の格納・復元の正確性、およびタイミングなしのケースでの後方互換性を保証する。

**ゴール**: `Set(key, samples, sampleRate, timings)` で格納したタイミング情報が `TryGet(key, out samples, out sampleRate, out timings)` で正確に復元されること、およびタイミングなしの `Set` → `TryGet` で timings == null が返ることを保証する。

## 2. 実装する内容の詳細

### テストファイル
`Assets/uPiper/Tests/Editor/AudioSynthesisCacheTests.cs`（既存ファイルに追加）

### 前提知識

P3-3 で AudioSynthesisCache に追加される変更:
- `CacheEntry` 構造体に `PhonemeTimingEntry[] Timings` フィールド追加（nullable）
- `Set` シグネチャに `PhonemeTimingEntry[] timings = null` パラメータ追加
- `TryGet` シグネチャに `out PhonemeTimingEntry[] timings` パラメータ追加（既存シグネチャは置き換え）

PhonemeTimingEntry の構造（P1-1 で定義）:
```csharp
public readonly struct PhonemeTimingEntry
{
    public string Phoneme { get; }
    public float StartSeconds { get; }
    public float EndSeconds { get; }
    public float DurationSeconds => EndSeconds - StartSeconds;
    internal PhonemeTimingEntry(string phoneme, float startSeconds, float endSeconds);
}
```

### テスト用ヘルパー

```csharp
/// <summary>
/// テスト用の PhonemeTimingEntry[] を構築する。
/// </summary>
private static PhonemeTimingEntry[] CreateSampleTimings()
{
    return new[]
    {
        new PhonemeTimingEntry("k", 0.0f, 0.058f),
        new PhonemeTimingEntry("o", 0.058f, 0.128f),
        new PhonemeTimingEntry("N", 0.128f, 0.186f),
    };
}
```

### 実装するテストメソッド

1. **Set_WithTimings_TryGetRestoresTimings**
   - Arrange: cache.Set(key, SampleAudio, 22050, CreateSampleTimings())
   - Act: cache.TryGet(key, out samples, out sampleRate, out timings)
   - Assert: timings != null
   - Assert: timings.Length == 3
   - Assert: timings[0].Phoneme == "k"
   - Assert: timings[0].StartSeconds == 0.0f (delta=1e-5f)
   - Assert: timings[0].EndSeconds == 0.058f (delta=1e-5f)
   - Assert: samples が SampleAudio と一致（既存動作の維持確認）

2. **Set_WithoutTimings_TryGetReturnsNullTimings**
   - Arrange: cache.Set(key, SampleAudio, 22050)（タイミングなし、既存の Set メソッド）
   - Act: cache.TryGet(key, out samples, out sampleRate, out timings)
   - Assert: timings == null
   - Assert: samples が SampleAudio と一致
   - Assert: sampleRate == 22050
   - 既存 API との後方互換性を検証

3. **Set_WithNullTimings_TryGetReturnsNullTimings**
   - Arrange: cache.Set(key, SampleAudio, 22050, timings: null)（明示的に null を渡す）
   - Act: cache.TryGet(key, out samples, out sampleRate, out timings)
   - Assert: timings == null
   - null を明示的に渡した場合と省略した場合の動作が一致することを検証

4. **Set_WithTimings_EvictsLRU_TimingsAlsoEvicted**
   - Arrange: maxEntries=2 のキャッシュを構築
   - key1 に timings 付きで Set
   - key2 に timings 付きで Set
   - key3 に timings 付きで Set（key1 が LRU 退避される）
   - Assert: cache.TryGet(key1, ...) == false（key1 が退避済み）
   - Assert: cache.TryGet(key2, ..., out timings2) == true、timings2 != null
   - Assert: cache.TryGet(key3, ..., out timings3) == true、timings3 != null
   - LRU 退避時にタイミング情報も正しく破棄されること

5. **Set_WithTimings_DuplicateKey_UpdatesTimings**
   - Arrange: 同一 key に対して timings 付きで Set → 別の timings で再 Set
   - Act: cache.TryGet(key, ..., out timings)
   - Assert: timings が最後に Set した値と一致（上書き成功）
   - Assert: cache.Count == 1

6. **Set_WithTimings_Clear_RemovesTimings**
   - Arrange: cache.Set(key, SampleAudio, 22050, CreateSampleTimings())
   - Act: cache.Clear()
   - Act: cache.TryGet(key, ..., out timings)
   - Assert: TryGet == false
   - Clear が timings も含めて正しくリセットすること

7. **TryGet_EmptyCache_TimingsIsNull**
   - Arrange: 空のキャッシュ
   - Act: cache.TryGet(999L, out samples, out sampleRate, out timings)
   - Assert: result == false
   - Assert: timings == null
   - Assert: samples == null
   - キャッシュミス時に timings が null であること

8. **Set_WithTimings_TryGetIgnoresTimingsViaDiscard**
   - Arrange: cache.Set(key, SampleAudio, 22050, CreateSampleTimings())
   - Act: cache.TryGet(key, out samples, out sampleRate, out _)（timings を破棄）
   - Assert: result == true
   - Assert: samples が SampleAudio と一致
   - タイミング付きで格納されたエントリの TryGet で timings を `out _` で無視しても正常動作すること

### コードパターン例

```csharp
[Test]
public void Set_WithTimings_TryGetRestoresTimings()
{
    // Arrange
    var cache = new AudioSynthesisCache();
    var key = AudioSynthesisCache.GenerateKey(
        SamplePhonemeIds, null, 1.0f, 0.667f, 0.8f, 0, 0);
    var timings = CreateSampleTimings();

    // Act
    cache.Set(key, SampleAudio, 22050, timings);
    var result = cache.TryGet(key, out var samples, out var sampleRate, out var restored);

    // Assert
    Assert.IsTrue(result);
    Assert.IsNotNull(restored, "タイミング情報が復元されること");
    Assert.AreEqual(3, restored.Length,
        "PhonemeTimingEntry の数が一致すること");
    Assert.AreEqual("k", restored[0].Phoneme);
    Assert.AreEqual(0.0f, restored[0].StartSeconds, 1e-5f);
    Assert.AreEqual(0.058f, restored[0].EndSeconds, 1e-5f);
    Assert.AreEqual(SampleAudio, samples);
}
```

## 6. 一から作り直すとしたら

テスト設計は「AudioSynthesisCache への timings 格納・復元の網羅」に絞るのが最適。既存の AudioSynthesisCacheTests.cs は LRU 退避・メモリ制限・統計カウンタ等を十分にカバーしているため、本チケットでは timings 関連の新機能のみテストする。

P3-3 は既存の `TryGet` シグネチャを `out PhonemeTimingEntry[] timings` パラメータ追加で置き換える設計。既存テストの呼び出し箇所には `out _` を追加する最小変更が必要だが、`Set` はデフォルトパラメータのため変更不要。テスト #8 で `out _` パターンの正常動作を検証する。

`PhonemeTimingEntry[]` は配列参照がキャッシュ内で共有されるが、PhonemeTimingEntry は readonly struct のため要素は不変。TTSSynthesisOrchestrator が `timings.ToArray()` で新しい配列を渡し、SynthesisWithTimingResult が防御的コピーを行うため、共有による問題は発生しない。

## 7. 後続タスクへの連絡事項

- P3-3 実装時: `CacheEntry` に `PhonemeTimingEntry[] Timings` フィールドを追加し、`TryGet` のシグネチャに `out PhonemeTimingEntry[] timings` パラメータを追加、`Set` のシグネチャに `PhonemeTimingEntry[] timings = null` パラメータを追加すること。既存の TryGet 呼び出し箇所には `out _` を追加する最小変更が必要
