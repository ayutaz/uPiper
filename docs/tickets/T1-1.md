# T1-1: TimingCalculator 基本計算テスト

## 1. タスク目的とゴール

TimingCalculator.Calculate() の基本的なタイミング計算ロジックが正確であることを検証する。piper-plus の TimingWriterTests.CalculateTiming_BasicDurations と同等の精度検証を NUnit ベースで実施し、frameLength 計算・累積時間・startSeconds/endSeconds の正確性を保証する。

## 2. 実装する内容の詳細

### テストファイル
`Assets/uPiper/Tests/Editor/AudioGeneration/TimingCalculatorTests.cs`

### テストクラス構成

```csharp
namespace uPiper.Tests.Editor.AudioGeneration
{
    [TestFixture]
    public class TimingCalculatorTests
    {
        private const int SampleRate = 22050;
        private const int HopSize = 256;
    }
}
```

### 共通ヘルパー

piper-plus の MakeMap() に相当する最小 phonemeIdMap を構築する。TTSSynthesisOrchestratorTests.CreateMinimalPhonemeIdMap() のパターンに準拠しつつ、タイミング計算に必要な PUA エントリも追加する。

**puaTokenMapper の扱い**: T1-1 の基本計算テストでは PUA 逆引きは不要なため、`Calculate()` の `puaTokenMapper` 引数には `null` を渡す。PUA 逆引きテストは T1-3 で `new PuaTokenMapper()` インスタンスを使用して実施する。

```csharp
private static Dictionary<string, int[]> CreateTimingTestPhonemeIdMap()
{
    return new Dictionary<string, int[]>
    {
        ["_"] = new[] { 0 },      // PAD
        ["^"] = new[] { 1 },      // BOS
        ["$"] = new[] { 2 },      // EOS
        ["a"] = new[] { 10 },
        ["k"] = new[] { 12 },
        ["\uE000"] = new[] { 17 }, // PUA for "a:"
        ["\uE019"] = new[] { 40 }, // PUA for "N_m"
    };
}
```

### 実装するテストメソッド

1. **Calculate_BasicDurations_ReturnsCorrectTiming**
   - 呼び出し: `TimingCalculator.Calculate(phonemeIds, durations, phonemeIdMap, puaTokenMapper: null, SampleRate, HopSize)`
   - 入力: phonemeIds = [10, 12], durations = [10.0f, 5.0f]
   - frameLength = 256 / 22050 ≈ 0.011609977f
   - 検証: entries[0] = ("a", start=0, end=10*frameLength)、entries[0].DurationSeconds == 10*frameLength（算出プロパティ）
   - 検証: entries[1] = ("k", start=10*frameLength, end=15*frameLength)、entries[1].DurationSeconds == 5*frameLength（算出プロパティ）
   - 精度: Assert.AreEqual with delta=1e-5f

2. **Calculate_BasicDurations_CumulativeTimeIsAccurate**
   - 呼び出し: `TimingCalculator.Calculate(phonemeIds, durations, phonemeIdMap, puaTokenMapper: null, SampleRate, HopSize)`
   - 入力: phonemeIds = [10, 12, 10], durations = [8.0f, 3.0f, 12.0f]
   - 検証: entries[2].EndSeconds == (8+3+12) * frameLength
   - 3音素の累積計算の正確性を保証

3. **Calculate_SinglePhoneme_ReturnsOneEntry**
   - 呼び出し: `TimingCalculator.Calculate(phonemeIds, durations, phonemeIdMap, puaTokenMapper: null, SampleRate, HopSize)`
   - 入力: phonemeIds = [10], durations = [7.0f]
   - 検証: entries.Length == 1, entries[0].StartSeconds == 0

4. **Calculate_ReturnsCorrectPhonemeStrings**
   - 呼び出し: `TimingCalculator.Calculate(phonemeIds, durations, phonemeIdMap, puaTokenMapper: null, SampleRate, HopSize)`
   - 入力: phonemeIds = [10, 12]
   - 検証: entries[0].Phoneme == "a", entries[1].Phoneme == "k"
   - phonemeIdMap の逆引きが正しく動作することを検証（puaTokenMapper=null でも通常音素の逆引きは動作する）

## 5. 懸念事項とレビュー項目

### 5.1 懸念事項

1. **浮動小数点精度**: frameLength の計算が float 除算のため、累積が大きくなると誤差が蓄積する可能性がある。delta=1e-5f で十分かの確認が必要
2. **piper-plus との API 差異**: piper-plus は `long[] phonemeIds` だが uPiper は `int[] phonemeIds`。テストコードでの型の違いに注意
3. **phonemeIdMap のキー形式**: PUA 文字をキーに持つエントリの扱い。TimingCalculator が BuildReverseIdMap で正しく処理するか

### 5.2 レビューチェックリスト

- [ ] NUnit の [TestFixture], [Test] 属性が正しく付与されているか

## 6. 一から作り直すとしたら

piper-plus の TimingWriterTests をそのまま NUnit に移植するのではなく、uPiper 固有の型（`int[]` phonemeIds、`PhonemeTimingEntry` readonly struct、`PuaTokenMapper` インスタンス注入）に最適化したテスト設計にする。全テストで `TimingCalculator.Calculate()` のシグネチャ（特に `puaTokenMapper` 引数）を正しく呼び出すことが重要。piper-plus の JSON/TSV シリアライズテスト3件は uPiper では非対応（TimingCalculator は計算のみ担当）のためスコープ外とする。
