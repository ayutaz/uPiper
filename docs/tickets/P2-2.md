# P2-2: InferenceAudioGenerator durations 読み取り

## 1. タスク目的とゴール

`InferenceAudioGenerator` にONNXモデルの `durations` 出力テンソル読み取り機能を追加する。モデルが2つ以上の出力テンソルを持つ場合、2番目の出力を `durations`（各音素のスペクトログラムフレーム数）として読み取り、`InferenceOutput` に格納して返す。

### なぜ必要か

- VITSモデルは `audio`（音声波形）と `durations`（音素フレーム持続時間）の2つの出力テンソルを持つ。現在のコードは1番目の出力（audio）のみ読み取り、`durations` を読み捨てている
- `durations` テンソルは `TimingCalculator`（P1-4）で秒単位のタイミング情報に変換され、最終的に `SynthesisWithTimingResult`（P1-3）として公開APIに露出する
- P2-1 で `GenerateAudioAsync` の戻り値が `InferenceOutput` に変更済みのため、`durations` を `InferenceOutput.Durations` に格納するだけで下流に伝搬できる

### 2.5 ExtractResults の変更

既存の `ExtractResults()`（L436-463）を `InferenceOutput` を返すように変更し、durations の読み取りロジックを追加する。

```csharp
/// <summary>
/// 推論結果を取得し、InferenceOutput として返す。
/// Audio は既存ロジックで抽出し、Durations はモデル対応時のみ抽出する。
/// </summary>
/// <remarks>Caller owns and must Dispose the returned InferenceOutput.</remarks>
private InferenceOutput ExtractResults()
{
    // --- Audio 抽出（既存ロジック） ---
    var audioData = ExtractAudioData();

    // --- Durations 抽出（新規） ---
    NativeArray<float> durations = default;
    if (_supportsDurations)
    {
        try
        {
            durations = ExtractDurationsData();
        }
        catch (Exception ex)
        {
            // durations 読み取り失敗は非致命的: Audio は返す
            PiperLogger.LogWarning(
                $"[InferenceAudioGenerator] Failed to read durations output: {ex.Message}. " +
                "Continuing without timing data.");
            if (durations.IsCreated)
                durations.Dispose();
            durations = default;
        }
    }

    return new InferenceOutput(audioData, durations);
}

/// <summary>
/// Audio 出力テンソルから NativeArray&lt;float&gt; を抽出する。
/// 既存 ExtractResults() のロジックをリファクタリングして分離。
/// </summary>
private NativeArray<float> ExtractAudioData()
{
    var outputTensor = GetOutputTensor();  // 既存メソッド
    var readableTensor = outputTensor.ReadbackAndClone();
    NativeArray<float> audioData = default;
    try
    {
        var audioLength = readableTensor.shape.length;
        audioData = new NativeArray<float>(audioLength, Allocator.Persistent);
        var src = readableTensor.DownloadToNativeArray();
        NativeArray<float>.Copy(src, audioData, audioLength);
        return audioData;
    }
    catch
    {
        if (audioData.IsCreated)
            audioData.Dispose();
        throw;
    }
    finally
    {
        readableTensor.Dispose();
    }
}

/// <summary>
/// Durations 出力テンソルから NativeArray&lt;float&gt; を抽出する。
/// Audio と同じ PeekOutput → ReadbackAndClone → DownloadToNativeArray パターン。
/// </summary>
private NativeArray<float> ExtractDurationsData()
{
    if (string.IsNullOrEmpty(_cachedDurationsOutputName))
        throw new InvalidOperationException(
            "Durations output name not cached. Ensure InitializeAsync completed.");

    var durationsTensor = _worker.PeekOutput(_cachedDurationsOutputName) as Tensor<float>;
    if (durationsTensor == null)
        throw new InvalidOperationException(
            $"Failed to get durations output '{_cachedDurationsOutputName}' from model");

    var readableTensor = durationsTensor.ReadbackAndClone();
    NativeArray<float> durations = default;
    try
    {
        var durLength = readableTensor.shape.length;
        durations = new NativeArray<float>(durLength, Allocator.Persistent);
        var src = readableTensor.DownloadToNativeArray();
        NativeArray<float>.Copy(src, durations, durLength);
        return durations;
    }
    catch
    {
        if (durations.IsCreated)
            durations.Dispose();
        throw;
    }
    finally
    {
        readableTensor.Dispose();
    }
}
```

### 2.6 ExecuteInference の変更

P2-1 で暫定的に `new InferenceOutput(audioData, default)` としていた箇所を、`ExtractResults()` が `InferenceOutput` を直接返すように変更する。

```csharp
private InferenceOutput ExecuteInference(...)
{
    var sw = Stopwatch.StartNew();
    using var ctx = PrepareInputs(phonemeIds, prosodyFlat,
        lengthScale, noiseScale, noiseW, speakerId, languageId);

    InferenceOutput output = default;
    try
    {
        var prepareMs = sw.Elapsed.TotalMilliseconds;

        sw.Restart();
        RunInference();
        var scheduleMs = sw.Elapsed.TotalMilliseconds;

        sw.Restart();
        output = ExtractResults();  // InferenceOutput を直接取得
        var extractMs = sw.Elapsed.TotalMilliseconds;

        var totalMs = prepareMs + scheduleMs + extractMs;
        PiperLogger.LogInfo(
            $"[InferenceAudioGenerator] Inference took {totalMs:F1}ms " +
            $"(prepare: {prepareMs:F1}ms, schedule: {scheduleMs:F1}ms, readback: {extractMs:F1}ms)" +
            $"{(output.HasDurations ? " [durations: available]" : "")}");

        return output;
    }
    catch (Exception ex)
    {
        output.Dispose();  // InferenceOutput.Dispose() は IsCreated チェック内蔵
        PiperLogger.LogError($"[InferenceAudioGenerator] Failed to execute inference: {ex.Message}");
        throw;
    }
}
```

### 2.10 設計判断の根拠

| 判断 | 理由 |
|------|------|
| Audio と Durations の抽出を別メソッドに分離 | 単一責任原則。テスタビリティ向上。Audio 抽出ロジックは既存コードのリファクタリング（動作変更なし） |
| durations 読み取り失敗を非致命的エラーとする | Audio は正常に生成されているため、durations 読み取り失敗で音声合成全体を失敗させるのは過剰。タイミング情報は付加価値であり、なくても音声は再生可能 |
| `Tensor<float>` として durations をキャスト | VITS モデルの durations テンソルは float32 型。Int 型の場合はキャスト失敗で `null` になり、非致命的エラーとして処理される |
| `_supportsDurations` フラグで判定 | `model.outputs.Count >= 2` のチェックを初期化時に1回だけ行い、推論ごとのチェックを回避 |
| 2番目の出力を durations と見なす | piper-plus の ONNX モデル仕様に従う。1番目: audio, 2番目: durations。出力名ではなくインデックスで判定（出力名はモデルごとに異なる可能性があるため） |

### 2.11 piper-plus との対応

| piper-plus | uPiper (P2-2) | 差分理由 |
|-----------|---------------|---------|
| `session.Run()` → `result[1]` で durations 取得 | `_worker.PeekOutput(_cachedDurationsOutputName)` | Unity Inference Engine API に合わせた呼び出し |
| `float[]` で durations を保持 | `NativeArray<float>` | Sentis API が NativeArray を返すため。managed コピー回避 |
| durations 読み取り失敗は例外送出 | 警告ログ + durations なしで続行 | Unity ランタイムの安定性を優先。音声再生は継続可能 |
| `SynthesisResult` record で返却 | `InferenceOutput` readonly struct で返却 | P1-2 の設計判断に従う |

## 6. 一から作り直すとしたら

### 代替案A: 出力名ではなくインデックスで durations を取得

```csharp
// PeekOutput(int index) オーバーロードがあれば
var durationsTensor = _worker.PeekOutput(1) as Tensor<float>;
```

**メリット**: 出力名キャッシュが不要。コードがシンプル。
**デメリット**: Unity.InferenceEngine の `Worker.PeekOutput()` は名前ベースのみ（インデックスオーバーロードが存在しない場合）。API 確認が必要。

**判定**: 名前ベースが確実。`_model.outputs[1].name` で名前を取得しキャッシュする方式を採用。

### 代替案B: durations を managed 配列 (float[]) で保持

```csharp
private float[] ExtractDurationsData()
{
    // NativeArray → float[] にコピー
    var durations = new float[durLength];
    readableTensor.DownloadToNativeArray().CopyTo(durations);
    return durations;
}
```

**メリット**: GC 管理で Dispose 不要。`InferenceOutput` の IDisposable が Audio のみ管理すればよい。
**デメリット**: P1-2 の設計判断（Audio と Durations の対称性維持）に反する。`NativeArray<float>.Copy` で managed コピーが発生。`TimingCalculator`（P1-4）が `float[]` を受け取る設計だが、`NativeArray<float>.ToArray()` で変換可能。

**判定**: P1-2 の設計に従い、`NativeArray<float>` で統一する。

### 代替案C: 初期化時に durations 対応を出力名で判定

```csharp
_supportsDurations = _model.outputs.Any(o => o.name == "durations");
if (_supportsDurations)
    _cachedDurationsOutputName = "durations";
```

**メリット**: 出力テンソルの名前が "durations" であることを明示的に検証。
**デメリット**: モデルによって出力名が異なる可能性がある（"durations", "dur", "predicted_durations" 等）。インデックスベースの方が汎用的。

**判定**: piper-plus の ONNX モデル仕様では2番目の出力が durations であるため、インデックスベース（`_model.outputs[1]`）で判定する。

- durations の結合は各句の durations 配列を連結し、句間の沈黙に対応するゼロ duration を挿入する

### P3-2 (TTSSynthesisOrchestrator タイミング計算統合) への連絡

- `InferenceOutput.Durations` の NativeArray<float> を `float[]` に変換して `TimingCalculator.Calculate()` に渡す: `output.Durations.ToArray()`
- `InferenceOutput.HasDurations` で durations 対応を判定
- durations の要素数は `phonemeIds.Length` と一致するはず。不一致時は `TimingCalculator` が `Math.Min` + 警告ログで処理する
