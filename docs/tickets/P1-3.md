# P1-3: SynthesisWithTimingResult 定義

## 1. タスク目的とゴール

音声合成結果とフォネムタイミング情報を一体で返却する公開APIの戻り値型 `SynthesisWithTimingResult` を定義する。既存の `PiperTTS.SynthesizeAsync()` が `AudioClip` 単体を返すのに対し、将来の `SynthesizeWithTimingAsync()` はこの型を返却してタイミング情報へのアクセスを提供する。

**ゴール**: `SynthesisWithTimingResult` クラスをコンパイル可能な状態で定義し、既存コードに副作用を与えないこと。

## 2. 実装する内容の詳細

### 2.1 新規ファイル

**パス**: `Assets/uPiper/Runtime/Core/AudioGeneration/SynthesisWithTimingResult.cs`

### 2.2 クラス設計

```csharp
using System;
using System.Collections.Generic;
using UnityEngine;

namespace uPiper.Core.AudioGeneration
{
    /// <summary>
    /// タイミング情報付き音声合成結果。
    /// <see cref="IPiperTTS.SynthesizeWithTimingAsync"/> の戻り値。
    /// AudioClip と各音素の開始・終了時刻を保持する。
    /// </summary>
    public sealed class SynthesisWithTimingResult
    {
        /// <summary>生成された AudioClip (22050Hz, float32)。</summary>
        public AudioClip AudioClip { get; }

        /// <summary>
        /// 各音素のタイミング情報。PAD/BOS/EOS を除く実音素のみ。
        /// モデルが durations テンソルを出力しない場合は null。
        /// </summary>
        public IReadOnlyList<PhonemeTimingEntry> Timings { get; }

        /// <summary>音声全体の長さ（秒）。AudioClip.length と等価。</summary>
        public float TotalDurationSeconds { get; }

        /// <summary>タイミング情報が利用可能かどうか。</summary>
        public bool HasTimings => Timings != null;

        /// <summary>
        /// SynthesisWithTimingResult を構築する。
        /// </summary>
        internal SynthesisWithTimingResult(
            AudioClip audioClip,
            IReadOnlyList<PhonemeTimingEntry> timings,
            float totalDurationSeconds)
        {
            AudioClip = audioClip
                ?? throw new ArgumentNullException(nameof(audioClip));
            // 防御的コピー: 外部からの変更を防ぐため配列をコピーして
            // AsReadOnly() で不変ラッパーを返す
            Timings = timings != null
                ? new List<PhonemeTimingEntry>(timings).AsReadOnly()
                : null;
            TotalDurationSeconds = totalDurationSeconds;
        }
    }
}
```

### 2.3 設計判断の根拠

| 項目 | 判断 | 理由 |
|------|------|------|
| sealed class vs struct | sealed class | `AudioClip` (参照型) と `IReadOnlyList<T>` (参照型) を含む。struct にすると防御的コピーの意味が薄れ、意図しない浅いコピーのリスクがある |
| internal コンストラクタ | PhonemizeResult パターン踏襲 | 外部パッケージからのインスタンス化を防止し、不正な状態のオブジェクト生成を回避する |
| IReadOnlyList<T> | List<T>.AsReadOnly() でラップ | 配列の防御的コピー + 不変ラッパーにより、呼び出し元からの変更を完全に防止。PhonemizeResult の Clone() パターンと同等の不変性保証 |
| Timings nullable | null 許容 | モデルが durations テンソルを出力しない場合（非対応モデル）に対応。HasTimings プロパティで判定可能 |
| TotalDurationSeconds | float | AudioClip.length と型を合わせる。秒単位は Unity の標準（AudioClip.length, AudioSource.time 等） |

### 2.4 namespace とアクセス修飾子

- namespace: `uPiper.Core.AudioGeneration` (同ディレクトリの既存型に準拠)

## 6. 一から作り直すとしたら

PhonemizeResult パターンの踏襲は適切な判断である。再設計するとしても以下の点は維持する:

- **sealed class**: AudioClip 参照を含むため struct は不適。sealed で継承を禁止し、型の契約を固定する
- **internal コンストラクタ**: 不正な状態のオブジェクト生成を防ぐ最もシンプルな手段
- **IReadOnlyList<T> + 防御的コピー**: 不変性の保証に必要。Array.AsReadOnly() でも可能だが、List<T>.AsReadOnly() の方がコンストラクタで List を new する流れと自然に合う

変更を検討する点:
- `TotalDurationSeconds` を `AudioClip.length` から算出する computed property にする案もあるが、AudioClip が Destroy 済みの場合に MissingReferenceException が発生するリスクがあるため、コンストラクタ時点で確定させる現設計が安全
