# T4-4: 統合テスト -- durations 非対応

## 1. タスク目的とゴール

SynthesizeWithTimingAsync() が durations 非対応モデル（StubGenerator の DurationsToReturn == null）の場合に Timings == null を返し、例外を投げずに AudioClip を正常に生成することを検証する。T3-3 で TTSSynthesisOrchestrator レベルの durations 非対応テストは実施済みだが、本チケットでは PiperTTS の公開 API レベル（SynthesizeWithTimingAsync()）での安全性を保証する。

**ゴール**: DurationsToReturn を default（null）に設定した StubGenerator 経由で SynthesizeWithTimingAsync() を呼び出し、Timings == null、HasTimings == false、例外なし、AudioClip 正常生成を保証する。

## 2. 実装する内容の詳細

### テストファイル
`Assets/uPiper/Tests/Editor/PiperTTSSynthesizeWithTimingTests.cs`（T4-1 で作成するファイルに追加）

### 前提知識

durations 非対応モデルのフォールバックパス:
1. StubGenerator.DurationsToReturn = null → InferenceOutput.HasDurations == false
2. TTSSynthesisOrchestrator.SynthesizeWithTimingCoreAsync() で TimingCalculator.Calculate() をスキップ
3. SynthesisWithTimingResult.Timings = null で構築
4. AudioClip は通常通り生成（durations の有無は AudioClip 生成に影響しない）

このフォールバックパスは T3-3 で TTSSynthesisOrchestrator レベルで検証済み。本チケットは同等のテストを公開 API レベル（PiperTTSSynthesizeWithTimingTests.cs 内）で実施し、TTSSynthesisOrchestrator.SynthesizeWithTimingAsync() 経由でのエンドツーエンド動作を保証する。

### 実装するテストメソッド

1. **SynthesizeWithTimingAsync_NoDurations_TimingsIsNull**
   - Arrange:
     - StubGenerator.DurationsToReturn = null（デフォルト状態、durations 非対応）
     - config: enableSilence = false
     - SynthesisRequest: phonemes = ["k", "o", "N"], prosodyFlat = null
   - Act: `orchestrator.SynthesizeWithTimingAsync(request)`
   - Assert: result.Timings == null
   - Assert: result.HasTimings == false

2. **SynthesizeWithTimingAsync_NoDurations_AudioClipNotNull**
   - Arrange: 同上（durations 非対応）
   - Act: `orchestrator.SynthesizeWithTimingAsync(request)`
   - Assert: result.AudioClip != null
   - Assert: result.TotalDurationSeconds > 0
   - durations がなくても AudioClip は正常に生成されること

3. **SynthesizeWithTimingAsync_NoDurations_NoException**
   - Arrange: 同上（durations 非対応）
   - Act & Assert: Assert.DoesNotThrowAsync(async () => await orchestrator.SynthesizeWithTimingAsync(request))
   - durations 非対応で例外が発生しないことを明示的に検証

4. **SynthesizeWithTimingAsync_NoDurations_GenerateCallCountIsOne**
   - Arrange: 同上（durations 非対応、句分割なし）
   - Act: `orchestrator.SynthesizeWithTimingAsync(request)`
   - Assert: _stubGenerator.GenerateCallCount == 1
   - durations の有無に関わらず推論は正常に実行されること

5. **SynthesizeWithTimingAsync_NoDurations_WithProsody_TimingsIsNull**
   - Arrange:
     - StubGenerator.DurationsToReturn = null、SupportsProsody = true
     - SynthesisRequest: phonemes = ["k", "o", "N"], prosodyFlat = [0,1,0, 1,1,0, 2,1,0]
   - Act: `orchestrator.SynthesizeWithTimingAsync(request)`
   - Assert: result.Timings == null
   - Assert: result.AudioClip != null
   - Prosody 付きでも durations 非対応なら Timings == null

6. **SynthesizeWithTimingAsync_NoDurations_CacheHit_TimingsStillNull**
   - Arrange:
     - StubGenerator.DurationsToReturn = null
     - AudioSynthesisCache インスタンスを生成
     - TTSSynthesisOrchestrator を audioCache 付きで構築
   - Act:
     - 1 回目: `orchestrator.SynthesizeWithTimingAsync(request)`（キャッシュ格納）
     - 2 回目: `orchestrator.SynthesizeWithTimingAsync(request)`（キャッシュヒット）
   - Assert: result2.Timings == null（キャッシュヒットでも Timings は null のまま）
   - Assert: result2.HasTimings == false
   - Assert: result2.AudioClip != null
   - durations 非対応モデルのキャッシュ格納/復元で Timings == null が維持されること

### コードパターン例

```csharp
[Test]
public async Task SynthesizeWithTimingAsync_NoDurations_TimingsIsNull()
{
    // Arrange: DurationsToReturn = null (デフォルト) → durations 非対応モデル
    var config = CreateValidatedConfig(enableSilence: false);
    var orchestrator = new TTSSynthesisOrchestrator(
        _stubGenerator, _splitOrchestrator, _phonemeEncoder, _audioClipBuilder,
        config, _voiceConfig, audioCache: null, puaTokenMapper: _puaTokenMapper);
    var request = new SynthesisRequest(
        new[] { "k", "o", "N" },
        null,
        1.0f, 0.667f, 0.8f,
        0, 0);

    // Act
    var result = await orchestrator.SynthesizeWithTimingAsync(request);

    // Assert
    Assert.IsNotNull(result, "SynthesisWithTimingResult が返ること");
    Assert.IsNull(result.Timings,
        "durations 非対応モデルでは Timings が null であること");
    Assert.IsFalse(result.HasTimings,
        "HasTimings が false であること");
}

[Test]
public async Task SynthesizeWithTimingAsync_NoDurations_CacheHit_TimingsStillNull()
{
    // Arrange: durations 非対応 + キャッシュ有効
    var cache = new AudioSynthesisCache();
    var config = CreateValidatedConfig(enableSilence: false);
    var orchestrator = new TTSSynthesisOrchestrator(
        _stubGenerator, _splitOrchestrator, _phonemeEncoder, _audioClipBuilder,
        config, _voiceConfig, audioCache: cache, puaTokenMapper: _puaTokenMapper);
    var request = new SynthesisRequest(
        new[] { "k", "o", "N" },
        null,
        1.0f, 0.667f, 0.8f,
        0, 0);

    // Act: 1回目 (miss) → 2回目 (hit)
    await orchestrator.SynthesizeWithTimingAsync(request);
    var result2 = await orchestrator.SynthesizeWithTimingAsync(request);

    // Assert
    Assert.AreEqual(1, _stubGenerator.GenerateCallCount,
        "2回目はキャッシュヒットで推論スキップ");
    Assert.IsNull(result2.Timings,
        "キャッシュヒットでも durations 非対応なら Timings は null");
    Assert.IsFalse(result2.HasTimings);
    Assert.IsNotNull(result2.AudioClip,
        "AudioClip はキャッシュから正常に復元されること");
}
```

## 6. 一から作り直すとしたら

テスト設計は「durations 非対応の安全性」に絞るのが最適。T3-3 で TTSSynthesisOrchestrator レベルの非対応テスト（7メソッド）は十分にカバーされているため、本チケットでは以下の追加価値に集中する:

1. **公開 API レベルの保証**: TTSSynthesisOrchestrator の内部実装が変更されても、SynthesizeWithTimingAsync() の外部契約（Timings == null、例外なし）が維持されること
2. **キャッシュとの相互作用**: durations 非対応モデルの結果がキャッシュに格納された場合でも Timings == null が維持されること（T3-3 ではキャッシュなしで検証）

テスト #6（キャッシュヒット後も null）が本チケットの最も独自な検証。T3-3 のテスト #1-#5 は TTSSynthesisOrchestrator レベルで対応済みだが、キャッシュとの相互作用は本チケットでのみ検証される。
