# 新言語Phonemizer C#実装戦略

## 言語別サマリ

| 言語 | 難易度 | C#行数見積 | 外部依存 | 推奨戦略 | 優先度 |
|------|--------|----------|---------|---------|--------|
| スペイン語 | **低** | ~900行 | なし | 直接ポート | **高** |
| フランス語 | 中 | ~750行 | なし | 直接ポート | **高** |
| ポルトガル語 | 中 | ~750行 | なし | 直接ポート | 中 |
| 中国語 | **高** | ~900行 | pypinyin相当 | ルックアップ+トーン規則 | 中 |
| 韓国語 | **高** | ~600行 | g2pk2相当 | Hangul分解+簡略規則 | 中 |
| 英語 | 高 | 既存 | Flite LTS | 既存実装活用 | 低(実装済み) |

## スペイン語（推奨: 直接ポート）

### 変換パイプライン
```
テキスト → 正規化 → トークン化 → グラフェム分割
→ 音節化 → ストレス検出 → ルールベースG2P → IPA音素列
```

### 主要な処理
- **グラフェム分割**: ch, ll, rr, qu, gu, gü を単位認識
- **音節化**: 強/弱母音、複合子音クラスタルール
- **ストレス検出**: アクセント記号 or 末尾規則
- **G2P**: 文脈依存ルール（b/d/g のアロフォン等）

### 音素インベントリ
- 母音: a, e, i, o, u
- 子音: p, b, t, d, k, ɡ, f, s, z, m, n, ɲ, l, ɾ, rr, tʃ, x, j, w, v
- アロフォン: β, ɣ, ð, ʝ
- ストレス: ˈ

### C#実装の利点
- 外部ライブラリ不要（純ルールベース）
- テストケースが充実（piper-plus側）
- 明確なルールセット

## フランス語（推奨: 直接ポート）

### 変換パイプライン
```
テキスト → 正規化 → トークン化 → コンテキスト依存G2P
→ ストレスマーカー挿入 → IPA音素列
```

### 主要な処理
- **鼻母音**: an/am→ɑ̃, en/em→ɛ̃, on/om→ɔ̃, in/im→ɛ̃
- **複合音**: eau→o, ouille→uj, tion→sjɔ̃
- **最終音声サイレント**: 末尾 d,g,h,m,n,p,s,t,x,z は消音
- **ストレス**: 最終音節

### 音素インベントリ
- 母音: a, e, i, o, u, y_vowel, ə, ø, œ
- 鼻母音: ɑ̃, ɛ̃, ɔ̃ (PUA: \ue056-\ue058)
- 子音: p, b, t, d, k, ɡ, f, v, s, z, ʃ, ʒ, m, n, ɲ, l, ɥ, j, w, ʁ

## ポルトガル語（推奨: 直接ポート）

### 変換パイプライン
```
テキスト → 正規化 → ストレス位置検出
→ G2P → 後処理（鼻母音化、L咽頭化、t/d変更） → IPA音素列
```

### 主要な処理
- **子音ダイグラフ**: nh→ɲ, lh→ʎ, ch→ʃ, rr→ʁ
- **鼻母音化**: 母音+n/m+子音 → 鼻化
- **後処理**: 重複鼻子音削除、L咽頭化(l→w)、t/d→tʃ/dʒ（ブラジルポルトガル語）

### 音素インベントリ
- 母音: a, e, i, o, u, ɛ, ɔ, ə + ã, ẽ, ĩ, õ, ũ
- 子音: p, b, t, d, k, ɡ, f, v, s, z, ʃ, ʒ, tʃ, dʒ, m, n, ɲ, l, ʎ, ɾ, ʁ, w, j

## 中国語（推奨: ルックアップテーブル + トーン規則）

### 変換パイプライン
```
テキスト → pypinyin（ピンイン変換）→ 正規化
→ トーン処理 → ピンイン→IPA変換 → トーンマーカー追加
```

### 課題
- **pypinyin**: Python外部パッケージ（C# NuGet相当なし）
- **多音字処理**: 文脈依存のピンイン決定
- **トーン連動規則**: 第3声+第3声→第2声等の4規則

### C#実装戦略
1. **ピンイン取得**: 事前計算データまたはルックアップテーブル
2. **ピンイン→IPA変換**: 固定マッピングテーブル（容易）
3. **トーン規則**: 4つの連動規則の実装（中程度）
4. **phonemize_from_pinyin_syllables API**: ピンイン列入力に対応

```csharp
public class ChinesePhonemizer : IPhonemizerBackend
{
    // ピンインINITIAL → IPA
    private static readonly Dictionary<string, string> InitialToIpa = new()
    {
        ["b"] = "p", ["p"] = "pʰ", ["d"] = "t", ["t"] = "tʰ",
        ["g"] = "k", ["k"] = "kʰ", ["j"] = "tɕ", ["q"] = "tɕʰ", // ...
    };

    // ピンインFINAL → IPA
    private static readonly Dictionary<string, string> FinalToIpa = new()
    {
        ["a"] = "a", ["ai"] = "aɪ", ["an"] = "an", ["ang"] = "aŋ", // ...
    };
}
```

## 韓国語（推奨: Hangul分解 + 簡略音韻規則）

### 変換パイプライン
```
テキスト → g2pk2（音韻規則適用）→ Hangul分解
→ 初声/中声/終声 → IPA変換
```

### 課題
- **g2pk2**: Python（MeCab依存）
- **音韻規則**: 連音化、鼻音化、格音化、経音化

### C#実装戦略
1. **Hangul分解**: Unicode演算で純C#実装可能（19×21×28テーブル）
2. **簡略音韻規則**: g2pk2なしで70-80%精度
3. **MeCab代替**: 困難 → 形態素解析なしの簡略版

```csharp
public static class HangulDecomposer
{
    // U+AC00..U+D7A3: 초성(19) × 중성(21) × 종성(28)
    public static (int initial, int medial, int final) Decompose(char syllable)
    {
        var code = syllable - 0xAC00;
        var initial = code / (21 * 28);
        var medial = (code % (21 * 28)) / 28;
        var final_ = code % 28;
        return (initial, medial, final_);
    }
}
```

## Prosody情報の言語別対応

| 言語 | A1 | A2 | A3 | 備考 |
|------|----|----|----|----|
| ja | モーラ位置 | アクセント核位置 | アクセント句位置 | 完全対応 |
| en | 0 | 0 | 0 | Prosodyなし |
| zh | tone(1-5) | 音節位置 | 単語長 | **非ゼロ値を計算**（ただしモデルのprosody_language_idsに含まれない場合ゼロ化） |
| ko | 0 | 0 | 音節数 | **A3は非ゼロ**（ただしモデル内でゼロ化される可能性） |
| es | 0 | 0 | 0 | Prosodyなし |
| fr | 0 | 0 | 0 | Prosodyなし |
| pt | 0 | 0 | 0 | Prosodyなし |

**注意**: 中国語・韓国語のPhonemizerは `phonemize_with_prosody()` で非ゼロ値を返すが、
現在のモデル（`prosody_language_ids={0}` = 日本語のみ）ではモデル内部でゼロ化される。
将来的に `prosody_language_ids` を拡張すれば、中国語のtone情報等を活用可能。

## 実装優先順位

1. **スペイン語** - 最も容易、外部依存なし（2-3週間）
2. **フランス語** - ルールベース（3-4週間）
3. **ポルトガル語** - ルールベース、後処理多段（3-4週間）
4. **中国語** - ピンイン辞書が必要（4-6週間）
5. **韓国語** - MeCab代替が課題（6-8週間）
